SECTION 1. C# ▼

What is the difference between an interface and an abstract class? Can a class implement multiple interfaces? What if two interfaces have the same method name?
Interface: pure contract, no implementation, multiple inheritance allowed, no constructor, interface IAnimal { void Speak(); }
use: Unrelated classes sharing same behavior; Ex. Payment service interface with pay method.
Abstract class: mix of abstract + concrete methods, only one inheritance allowed, can have constructor, abstract class Shape { public abstract void Draw(); }, use: Closely related classes shared base logic with enforced contract; Ex. Policy abstract class with some common logic.
Normal class (base class): regular class with full implementation, object can be created, child classes inherit and reuse its logic & do not need to change behavior, class Logger { public void Log(string m) { } }
Same method name conflict: use explicit interface implementation, each gets own body, called via specific interface reference, void IAnimal.Speak() { } void IHuman.Speak() { }
What is the difference between a concrete class and an abstract class?
Concrete class: fully implemented, all methods defined, can be instantiated, var obj = new MyClass();, use: creating actual objects
Abstract class: partial implementation, cannot be instantiated, forces child classes to implement abstract methods, use: shared base + contract
Types of constructors? What is a private constructor? How many static constructors can a class have?
Types: Default (no params), Parameterized, Copy, Static, Private
Private constructor: prevents external instantiation, use: Singleton pattern, factory methods
Static constructor: only one allowed, no params, runs once automatically before first use, use: one-time static initialization
What is the Singleton pattern? How do you make it thread-safe?
Singleton: only one instance exists throughout app lifetime, private constructor prevents external instantiation, use: config service, logging, caching
Thread-safe: safe access from multiple threads, prevents race conditions, use lock(obj) for critical sections, Lazy<T> for thread-safe lazy initialization, or concurrent collections like ConcurrentDictionary
Why use a Singleton class instead of a static class?
Singleton: single instance, supports DI, interfaces, polymorphism, testable & replaceable, use: config, cache, services
Static: no instance, no DI, no interfaces, no polymorphism, tightly coupled, use: utility methods
What is Dependency Injection (DI)? What is Inversion of Control (IoC)? What are the DI life times? What are the Injection types?
IoC: Creating dependencies given to external system, not the class itself, use: Loosely coupled design
DI: most common IoC implementation, class receives dependencies from outside instead of creating manually. Register in program.cs
Lifetimes:
Transient – new instance per each use, use: lightweight, stateless services (helpers, validators)
Scoped – one instance per HTTP request, use: stateful services (DbContext, repository)
Singleton – single instance for entire app lifetime, use: shared services like caching, configuration
Types: Constructor injection (most common, preferred), Property injection, Interface injection, Service Locator (anti-pattern, avoid)
What is the Interface Segregation Principle (ISP)?
ISP: don't force a class to implement methods it doesn't need, split large interfaces into smaller focused ones, IWorkable + IFeedable instead of one big IWorker, use: keeps classes clean, avoids empty or throw implementations
What is the difference between value types and reference types? How does Stack and Heap relate to them?
Value types: store actual value, copied on assignment, usually stored in Stack, fast allocation, examples: int, double, bool, struct
Reference types: store reference (memory address) to object, object stored in Heap, reference copied not object, examples: class, string, array, object
Stack: stores value types and references, fixed size, fast LIFO memory, automatically managed
Heap: stores objects of reference types, dynamic size, managed by Garbage Collector (GC)
What is boxing and unboxing in C#?
Boxing: converting value type to reference type (object), value copied from Stack to Heap, example: int x = 10; object obj = x;
Unboxing: converting reference type (object) back to value type, requires explicit cast, value copied from Heap to Stack, example: int y = (int)obj;
note: boxing/unboxing causes performance overhead due to heap allocation and casting
What is Garbage Collection in .NET?
GC: automatically frees heap memory of unreferenced objects, no manual memory management needed
Generations: Gen 0 (short-lived, collected frequently), Gen 1 (survived one collection), Gen 2 (long-lived objects)
Note: GC handles managed memory only, implement IDisposable for unmanaged resources like DB connections, file handles
What is IDisposable and the using statement?
IDisposable: interface with Dispose(), implement when class holds unmanaged resources like DB connections, file handles, streams, use: manual resource cleanup
using statement: auto-calls Dispose() when block ends, even on exception, prevents resource leaks, using var conn = new SqlConnection();
What is the difference between Task, Thread, and Async/Await? What is Thread Pool and how does TPL work?
Thread: Worker, OS level, heavy, manual management, expensive to create/destroy, use: rarely used directly
Thread Pool: managed pool of reusable threads by .NET, avoids create/destroy overhead, tasks run on pool threads by default
Task: Work, lightweight unit of work on thread pool, supports cancellation and continuation, Task.Run(() => DoWork()), use: CPU-bound background work
TPL: Task Parallel Library, provides Parallel.For Parallel.ForEach Task.WhenAll, use: CPU-heavy parallel workloads
async/await: frees thread while waiting, doesn't block, await GetDataAsync(), use: I/O-bound work like DB/HTTP calls
How does async/await work under the hood? What is ConfigureAwait(false)?
async/await: on hitting await, method pauses and returns control to caller without blocking thread, resumes when task completes, compiler converts to state machine under the hood
ConfigureAwait(false): resume on any thread, not original context, avoids deadlocks, use: library or non-UI code
What is Parallel programming? Difference between Parallel.For, Parallel.ForEach, and Task.WhenAll?
Multithreading: multiple threads executing different tasks concurrently, tasks may run simultaneously or sequentially use: handling different operations at the same time (UI, I/O, background work)
Parallelism: multiple threads performing the same or similar work simultaneously, task is split into smaller parts and executed across CPU cores, use: speeding up CPU-bound workloads (data processing, calculations)
Parallel.For: parallel loop over index range, Parallel.For(0, 10, i => Process(i));, use: fixed range CPU work
Parallel.ForEach: parallel loop over collection, Parallel.ForEach(list, item => Process(item));, use: collection CPU work
Parallel.Invoke: runs multiple independent actions in parallel, Parallel.Invoke(()=>Task1(),()=>Task2());, use: execute independent methods simultaneously
Task.WhenAll: runs multiple async tasks concurrently, waits for all task to complete, await Task.WhenAll(task1, task2);, use: multiple independent I/O calls
Thread safety: use ConcurrentBag<T> or lock when threads share data
What is the difference between var, dynamic, and object?
var: compile-time type inference, strongly typed, type fixed after assignment, var x = 5 = int x = 5, use: LINQ, complex type names
dynamic: runtime type resolution, no compile-time checking, errors only at runtime, use: COM interop, reflection, avoid in regular code
object: base type of all types, holds anything but needs explicit casting, compile-time checked, use: when type is truly unknown
What is the difference between readonly and const?
const: compile-time constant, assigned at declaration only, implicitly static, primitives/string/enum only, const int PI = 3.14;, use: fixed known values like PI, config constants
readonly: runtime constant, assigned at declaration or constructor, any type allowed, per-instance or static, readonly int _studentId, use: values known only at runtime or per instance
What is the difference between value, ref, and out parameters?
value: copy passed, original untouched, default behavior, use: when caller's variable should not change
ref: passes original variable, must be initialized before passing, changes reflect in caller, ref int x, use: method needs to read and modify caller's variable
out: passes original variable, no need to initialize before passing, method must assign before returning, int.TryParse("5", out int result), use: returning multiple values from a method
What is the difference between == and .Equals()?
== operator: compares values for value types, compares references (memory address) for reference types by default, can be overloaded by classes
.Equals(): method used to compare object equality (logical/value equality), can be overridden to define custom comparison logic
example: object a = "hello"; object b = "hello"; a == b compares references, a.Equals(b) compares values
What is the difference between IEnumerable and IQueryable?
IEnumerable: works on in-memory collections, query executed in application memory, LINQ-to-Objects, filtering happens after data is loaded, use: when data already loaded in memory. example: var users = context.Users.ToList();
IQueryable: works with remote data sources (database), builds expression tree and converts query to SQL, filtering happens at database level, use: efficient querying with large datasets example: context.Users.Where(x => x.Age > 30) → SQL generated with IQueryable
What is the difference between First, FirstOrDefault, Single, and SingleOrDefault?
First(): returns first match, throws if none found, use: when result is guaranteed to exist
FirstOrDefault(): returns first match, returns null if none, use: safe default choice in most cases
Single(): expects exactly one match, throws if none or more than one, use: unique lookup where duplicates are a bug
SingleOrDefault(): returns null if none, throws if more than one, use: unique lookup that may not exist
What is the difference between Dictionary and HashSet?
Dictionary<K,V>: key-value pairs, lookup value by key, keys must be unique, O(1) lookup, dict["key"] = value;, use: mapping/lookup by key
HashSet<T>: unique values only, no key-value pair, O(1) lookup, set.Contains(x);, use: membership check, removing duplicates
What are Delegates in C#? What is the difference between Action, Func, and Predicate?
Delegate: type-safe function pointer, pass method as argument to another method, holds reference to method, supports singlecast & multicast via +=, use: callbacks, events, LINQ
example: payment system choosing payment method dynamically, ProcessPayment(PayWithGPay), ProcessPayment(PayWithPhonePe), ProcessPayment(PayWithPaytm)
Action: built-in delegate, returns void, Action<string> a = msg => Console.WriteLine(msg);, use: operations with no return value
Func: built-in delegate, returns value, last type param is return type, Func<int,int,int> f = (a,b) => a+b;, use: LINQ, transformations
Predicate: delegate returning bool, Predicate<int> p = x => x > 10;, use: filtering/conditions in collections
Event delegate: delegate with safety wrapper, only raised from declaring class, use: publisher–subscriber pattern
example: button click event notifying multiple subscribers, e.g., show message, log action, send notification
What are Generics and why are they useful?
Generics: write type-safe code that works with any type without duplication, no casting needed, no boxing/unboxing overhead, List<T> Dictionary<K,V>, use: reusable collections, repositories, utility methods
What are Extension Methods?
Extension Methods: add methods to existing types without modifying or inheriting them, defined in static class, first param uses this keyword, public static int WordCount(this string s), use: extend third-party or built-in types, all LINQ methods are extension methods
What is the difference between String and StringBuilder?
string: immutable, every modification creates new object in memory, use: few or no modifications
StringBuilder: mutable, modifies same object in place, much faster for repeated changes, sb.Append("text");, use: loops with many string concatenations
What is Pattern Matching in C#?
Pattern matching: check type/value/object structure and extract data in one step, reduces casting and complex conditions
Type pattern: if(obj is string s)
Constant pattern: if(x is 10)
Relational pattern: if(age is > 18)
Property pattern: if(p is { Age: > 18 })
Switch pattern: x switch { <0 => "Neg", >0 => "Pos" }

SECTION 2. ASP.NET Core / Web API ▼

What is ASP.NET MVC architecture? What are its components?
MVC: separates app into 3 layers, clean separation of concerns
Model: data and business logic, C# classes, EF entities, DTOs
View: UI layer, Razor pages or Angular templates
Controller: handles requests, calls services, returns view or JSON, use: entry point for every HTTP request
What is the difference between Program.cs and Startup.cs?
Startup.cs (.NET 5 and below): two methods — ConfigureServices() for DI registration, Configure() for middleware pipeline
Program.cs (.NET 6+): Startup.cs removed, everything in one file, services via builder.Services.Add...(), middleware via app.Use...(), cleaner minimal hosting model. Entry point of ASP.NET Core app, configures services and middlewares
What is middleware and how does the request pipeline work? What is the difference between app.Use(), app.Run(), and app.Map()?
Middleware: components that handle request/response in pipeline, each can process request, call next, then process response on return
Common Middleware Order: UseExceptionHandler → UseStaticFiles → UseRouting → UseCors → UseAuthentication → UseAuthorization → MapControllers
app.Use(): adds middleware, can call next middleware using next delegate, used for most cases, use: logging, auth, exception handling
app.Run(): terminal middleware, stops pipeline, no next called, usually at end, use: final response handler
app.Map(): branches pipeline based on URL path, use: different handling for specific routes
What are the three service lifetimes — Transient, Scoped, Singleton?
Transient: new instance every time requested, use: lightweight stateless services
Scoped: one instance per HTTP request, use: DbContext, request-specific logic
Singleton: one instance for entire app lifetime, use: caching, logging, configuration services
How does JWT authentication work in Web API?
Flow: user login → server validates credentials → server generates JWT token → client stores token in HttpOnly cookie → client sends token in every request
Header: Authorization: Bearer <token>
Token Parts: Header (algorithm), Payload (claims/user data), Signature (verification)
Validation: server verifies signature, expiry, issuer, audience before allowing access. AddAuthentication().AddJwtBearer(), [Authorize]
What is OAuth? What are Access Token and Refresh Token? What is the difference between OAuth and JWT?
OAuth 2.0: authorization framework which generates tokens, lets user grant third-party app access without sharing password, use: Login with Google/Facebook
JWT: token format, compact signed string with claims, OAuth uses JWT as token format but JWT can be used independently too
Access Token: short-lived (minutes/hours), sent in Authorization header, use: access protected APIs
Refresh Token: long-lived, stored in HttpOnly cookie, used to get new access token without re-login, never sent to API directly
How do you secure an API? How do you handle security vulnerabilities?
Auth: JWT with short expiry, [Authorize] on all protected endpoints, HTTPS only
CORS: restrict allowed origins, builder.Services.AddCors() + app.UseCors()
SQL Injection: use parameterized queries or EF Core, never concatenate user input into SQL
XSS: Angular auto-escapes by default, avoid innerHTML @HTML.Raw with user data
CSRF: use anti-forgery tokens for form submissions
Secrets: store in Azure Key Vault, never in code
Other: input validation, rate limiting, never expose sensitive data in responses or URL, Encrypt policyNumber
What is CORS and how do you configure it?
CORS (Cross-Origin Resource Sharing): browser security policy that blocks requests from different origin (protocol/domain/port)
Purpose: allow frontend (different domain) to call Web API
Configure: define CORS policy in services, then enable middleware
Example: AddCors() → define allowed origins, headers, methods → UseCors()
Options: AllowAnyOrigin, AllowAnyHeader, AllowAnyMethod, WithOrigins(specific domains)
What is the difference between IActionResult, ActionResult, and specific return types?
IActionResult: base HTTP response type, most flexible, Use when no specific model and returning multiple HTTP responses (Ok(), NotFound(), BadRequest())
ActionResult<T>: use when returning specific data model + status codes, allows returning object directly, better Swagger support, use: most common in Web APIs example: return user; or return NotFound();
Specific type (e.g. User): returns only data, no control over HTTP status codes, use: simple read-only endpoints example: public User GetUser()
What is the difference between PUT and PATCH? How do you handle PATCH for updating a specific field like email?
PUT: replaces entire resource, send full object, use: full update
PATCH: partial update, send only changed fields, use: update single field like email
PATCH implementation: PatchDTO + AutoMapper conditional mapping (srcMember != null)
How do you return custom HTTP status codes from a Web API?
Built-in helpers: return Ok(data) (200), return NotFound() (404), return BadRequest("msg") (400), return StatusCode(500, "error") for custom codes, all return IActionResult
What is Idempotency? Why does it matter in APIs?
Idempotent: calling same operation multiple times produces same result as calling once
GET, PUT, DELETE: idempotent — POST: not idempotent, calling twice creates two records
Why it matters: network timeouts cause retries, idempotent endpoints won't create duplicate data
Implementation: idempotency keys — client sends unique request ID, server checks if already processed
How do you perform input validation? What is FluentValidation?
Data Annotations: [Required], [StringLength], [Range], [EmailAddress], [MaxLength], check ModelState.IsValid in controller,
Custom Validation: create custom rules using ValidationAttribute or implement IValidatableObject for model-level validation. FluentValidation: External library, validation rules in separate class, clean readable syntax, better for complex rules, register with AddFluentValidation() in Program.cs, use: complex conditional validations
How do you implement global exception handling in ASP.NET Core? Why would you still use try-catch inside a controller?
Global handling: custom middleware wrapping pipeline in try-catch, register first so it catches all exceptions, or use built-in app.UseExceptionHandler() or IExceptionFilter
try-catch in controller: use when specific error is expected, proper log with error code, path and message instead of generic error , e.g. catch DbUpdateException to return clear 400 instead of generic 500
What are Filters in ASP.NET Core? Filter types, How are they different from middleware?
Filters: attributes that runs around controller action methods in MVC pipeline
Filter Types:
Authorization Filter: runs first before all filters before action, checks user authorization, use: role/policy validation
Resource Filter: runs before model binding and after the action result use: caching, performance logic, should we continue this request?
Action Filter: runs before/after controller action, use: logging, validation
Result Filter: runs before/after action result execution, use: modify response/result
Exception Filter: handles exceptions from controller/action, use: global error handling
Difference from Middleware: middleware runs for all HTTP requests in pipeline, filters run only inside MVC pipeline for controller actions
What is API caching and how can it be implemented?
Caching: stores response data so server doesn't recompute on every request
Response caching: [ResponseCache] attribute, use: simple endpoint-level caching
In-memory cache: IMemoryCache, use: single server apps
Distributed cache: Redis, use: multi-server/load-balanced setups
What is API versioning and how do you implement it? What is Backward compatibility?
API versioning: Multiple APIs version for different client, release API changes without breaking existing clients
Approaches: URL versioning /api/v1/users (most common), header versioning, query string ?api-version=1.0
Implementation: Asp.Versioning.Mvc NuGet package, configure in Program.cs
Backward compatibility: keep old clients working by using API versioning, adding fields instead of changing/removing them, optional parameters, and gradual API deprecation.
What is content negotiation in Web API? How do you restrict it to specific formats?
Content negotiation: client requests response format via Accept header, server responds accordingly, default is JSON in ASP.NET Core
Restrict to JSON: don't register AddXmlSerializerFormatters(), or use [Produces("application/json")] on specific action
What is state management? Data passing techniques in MVC?
State management: ViewData, ViewBag, TempData, Session, Cookies, Hidden Fields, Query Strings
Data passing: Model, ViewData, ViewBag, TempData, Form
How do you log errors in ASP.NET Core?
Built-in: inject ILogger<T>, use _logger.LogError() LogWarning() LogInformation(), providers: Console, Debug, EventLog
Production: use Serilog (write to files, DB, cloud) or NLog, use: structured logging with Application Insights or Seq
Which unit testing framework do you use and how do you mock data?
Framework: xUnit (preferred in .NET Core) or NUnit
Moq: creates fake versions of dependencies, test class in isolation, var mock = new Mock<IUserRepository>(); then mock.Setup(r => r.GetById(1)).Returns(fakeUser);
use: test only business logic, not the actual DB, Use wiremocks for integration tests

SECTION 2.1. Microservices ▼

What are Microservices? How are they different from Monolithic architecture?
Microservices: architecture where application is split into small independent services, each service handles a specific business capability and communicates via APIs; Independent deploy(docker), scale per service, separate db, different tech per service, easier maintenance
Monolithic: entire application built as single unit with UI, business logic, and data access in one codebase; Single deploy, scale whole app, single tech stack, harder maintenance, big failure
How do microservices communicate with each other?
Synchronous (HTTP/REST or gRPC): service calls another and waits, simple but tightly coupled, caller fails if target is down, use: real-time queries needing immediate response
Asynchronous (Message Queue): service publishes message to queue storage and subscriber services listen to it, fully decoupled, sender doesn't care if receiver is down, Azure Service Bus / RabbitMQ / Kafka, use: events, notifications, background processing
What is an API Gateway? Why is it important?
API Gateway: single entry point for all client requests, clients never call individual services directly, handles routing, auth, rate limiting, load balancing, logging in one place
Examples: AWS API Gateway, Azure API Management, Ocelot (.NET)
use: enforce security in one place, change backend services without affecting clients
What is the Saga Pattern? How do you handle distributed transactions across microservices? What happens if 2 succeed and last fail?
Saga: executes operations step-by-step across multiple services, if step fails previous steps undone via compensation actions, like SQL transaction but for distributed systems, Order → Payment → Inventory
Choreography: services communicate via events, no central controller, loosely coupled but harder to track, use: Azure Service Bus
Orchestration: central orchestrator coordinates each step, easier to follow but orchestrator is a dependency, use: Azure Durable Functions
Partial failure → retry (Polly), send failed messages to DLQ, use messageId for idempotency (duplicate detection),
apply Saga pattern: Each service has success actions & compensating actions triggered via different events
What is the Circuit Breaker pattern? Why is it important?
Circuit Breaker: stops retrying a failing service, prevents cascading failures across system
States: Closed (normal), Open (failures exceeded threshold, returns fallback immediately), Half-Open (tries one request to check recovery)
Implementation: Polly library in .NET, supports retry, circuit breaker, timeout policies, use: any service-to-service HTTP call
How do you trace requests across multiple microservices? What did you use in Aviva?
Distributed Tracing: unique Correlation ID generated at entry point, passed in HTTP header to every downstream service, each service logs this ID, filter all logs for single request across all services
Aviva: implemented correlation ID logging across 30+ services, reduced debugging time from 2–3 days to ~2 hours
What is Service Discovery in microservices?
Service Discovery: mechanism where services register in a service registry and other services discover them dynamically instead of using hardcoded URLs. Use: avoids hardcoded URLs since service IP/addresses can change in multi server auto-scaling.
Client-side discovery: calling service queries registry (e.g., Consul, Eureka) to get service address, then calls it directly.
Server-side discovery: request goes through load balancer/API Gateway which looks up the service and routes the request.
What are the main challenges of microservices and how do you handle them?
Distributed tracing: hard to debug across services → Correlation IDs + Serilog + Application Insights
Data consistency: no shared DB, eventual consistency → Saga pattern
Service failures: one failing service can cascade → Circuit Breaker with Polly, retry with exponential backoff
Network latency: more calls than monolith → async messaging, caching
Deployment complexity: many services to manage → Docker + Kubernetes, CI/CD pipelines

SECTION 3. Entity Framework Core ▼

What is DbContext and DbSet?
DbContext: main class which represents database, manages DB connection and all operations (query, save, track changes), Scoped lifetime
DbSet<T>: represents a table, each property maps to one table, query through DbSet, commit via SaveChanges()
What is the difference between Code First and Database First?
Code First: write C# entity classes, EF generates DB from them, use migrations for schema changes, use: new projects, full control from code
Database First: DB already exists, EF scaffolds C# models from it, use: existing legacy databases
What are EF Core migrations?
Migrations: Changes made in entity apply to DB incrementally, like version control for DB schema
Add-Migration MigrationName creates migration file, Update-Database applies it, use: every time model/schema changes
What is the difference between Lazy Loading and Eager Loading?
Eager Loading: fetches main entity and related data together using .Include() in one query (JOIN), use: when you know related data is required
Lazy Loading: related data is fetched later only when the navigation property is accessed, triggers additional queries (can lead to N+1 issue), use: generally avoided in performance-sensitive APIs. Always include related data you know you'll need.
What is AsNoTracking() and when should you use it?
AsNoTracking(): Tells EF Core not to track retrieved entities in the change tracker, improves performance and reduces memory usage, use: read-only queries where data will not be updated.
How do you optimize EF Core queries?
AsNoTracking(): for all read-only queries
.Select(): fetch only needed columns, avoid loading full entity
.Include(): avoid N+1, load related data upfront
IQueryable: keep query as IQueryable as long as possible, filters run as SQL not in memory
Pagination: use Skip/Take, never load all rows
Indexes: add proper DB indexes on filtered/joined columns
Compiled queries: precompile frequently executed queries to reduce query translation overhead and improve performance.
How do you configure a Many-to-Many relationship in EF Core?
Define navigation collections in both entities and configure using
modelBuilder.Entity<Student>().HasMany(s => s.Courses).WithMany(c => c.Students)
With junction table: needed when extra columns required (e.g. enrollment date), create StudentCourse entity with composite key StudentId + CourseId, configure two one-to-many relationships

SECTION 4. Design Patterns & Architecture ▼

What are the SOLID principles?
S — Single Responsibility: class should have only one reason to change, one job only, UserService handles user logic — not emails too
O — Open/Closed: open for extension, closed for modification, add new behavior via new class not by changing existing code
L — Liskov Substitution: child class should be replaceable for parent without breaking app, Penguin should not inherit Bird.Fly() since penguins can't fly
I — Interface Segregation: don't force class to implement methods it doesn't need, split large interfaces into smaller focused ones
D — Dependency Inversion: high-level classes depend on abstractions not concrete implementations, Controller depends on IUserService not UserService
Which design patterns have you used? Explain each.
Singleton: one shared instance throughout app, use: config service, logging, caching
Repository: abstracts DB access from business logic, easier to test and swap DBs, use: data access layer
Factory: creates objects without exposing creation logic, use: when object creation is complex or conditional
Strategy: swap algorithms at runtime without changing client code, use: payment methods, sorting strategies
Decorator: add behavior to objects dynamically without modifying them, use: logging, caching wrappers
CQRS: separates read and write operations into different models/services, use: complex systems, performance and scalability

What are SQL Constraints? What is the difference between a primary key and a unique key?
Constraints: rules enforced on columns to maintain data integrity; Validate user input

Consistency: DB stays in valid state before and after

What are deadlocks in SQL? How do you prevent them?

TRY/CATCH: wrap logic in BEGIN TRY...END TRY / BEGIN CATCH...END CATCH

What are indexes? How do they help and what are the downsides?

How do you optimize SQL queries?

1. Reverse a string in place.
Loop from end of string and build the reversed result. 

// Option1: Reverse string using for reverse loop
string ReverseFor(string s)
{
    string result = "";
    for (int i = s.Length - 1; i >= 0; i--)
    {
        result += s[i];
    }
    return result;
}

// Option2: Reverse string using LINQ
string ReverseLinq(string s)
{
    return new string(s.Reverse().ToArray());
}
2. Reverse the words in a sentence.
Split by space, reverse the array, join back. 
string ReverseWords(string s)
{
    string[] words = s.Split(' ');
    string[] result = new string[words.Length];

    for (int i = 0; i < words.Length; i++)
    {
        result[i] = words[words.Length - 1 - i];
    }

    return string.Join(' ', result);
}

    {
        rev += s[i];
    }
    return s == rev;
}

2. Write a thread-safe Singleton class in C#.
Use lock with double-null check — thread safe, creates instance only when first accessed. 
public sealed class Singleton
{
    private static Singleton _instance;
    private static readonly object _lock = new object();

    private Singleton() { }

    public static Singleton Instance
    {
        get
        {
            lock (_lock)
            {
                if (_instance == null)
                    _instance = new Singleton();

                return _instance;
            }
        }
    }

5. How to pass list to stored procedure using EF Core?
Use TVP: create user-defined table type in SQL, convert list to DataTable, pass as structured SqlParameter. 

-- SQL (Type + Stored Procedure)
CREATE TYPE UserListType AS TABLE (
    UserId INT,
    UserName NVARCHAR(100)
);

CREATE PROCEDURE GetUsersData
    @Users UserListType READONLY
AS
BEGIN
    SELECT *
    FROM Users u
    INNER JOIN @Users ul ON u.UserId = ul.UserId;
END


// EF Core + TVP
[HttpPost]
public IActionResult GetUsers()
{
    var users = new List<UserType>
    {
        new UserType { UserId = 1, UserName = "A" },
        new UserType { UserId = 2, UserName = "B" }
    };

    var table = new DataTable();
    table.Columns.Add("UserId", typeof(int));
    table.Columns.Add("UserName", typeof(string));

    foreach (var u in users)
        table.Rows.Add(u.UserId, u.UserName);

    var param = new SqlParameter
    {
        ParameterName = "@Users",
        SqlDbType = SqlDbType.Structured,
        TypeName = "dbo.UserListType",
        Value = table
    };

    var result = _context.Users
        .FromSqlRaw("EXEC GetUsersData @Users", param)
        .ToList();

    return Ok(result);
}
6. How to call multiple APIs in Angular?

    });
  }
}
