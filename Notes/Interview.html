SECTION 1. C# ▼

What is the difference between an interface and an abstract class? Can a class implement multiple interfaces? What if two interfaces have the same method name?
Interface: pure contract, no implementation (before C# 8), multiple allowed, no constructor, interface IAnimal { void Speak(); }, use: unrelated classes sharing same behavior
Abstract class: mix of abstract + concrete methods, only one allowed, can have constructor, abstract class Shape { public abstract void Draw(); }, use: shared base logic with enforced contract
Same method name conflict: use explicit interface implementation, each gets own body, called via specific interface reference, void IAnimal.Speak() { } void IHuman.Speak() { }
What is the difference between a concrete class and an abstract class?
Concrete class: fully implemented, all methods defined, can be instantiated, var obj = new MyClass();, use: creating actual objects
Abstract class: partial implementation, cannot be instantiated, forces derived classes to implement abstract methods, use: shared base with enforced contract
Types of constructors? What is a private constructor? How many static constructors can a class have?
Types: Default (no params), Parameterized, Copy, Static, Private
Private constructor: prevents external instantiation, use: Singleton pattern, factory methods
Static constructor: only one allowed, no params, runs once automatically before first use, use: one-time static initialization
What is the Singleton pattern? How do you make it thread-safe?
Singleton: only one instance exists throughout app lifetime, private constructor prevents external instantiation, use: config service, logging, caching
Thread-safe: use Lazy<T>, handles thread safety automatically, instance created only on first access, private static readonly Lazy<T> _instance = new(() => new T());
What is Dependency Injection (DI)? What is Inversion of Control (IoC)? What are the types?
IoC: control of creating dependencies given to external system, not the class itself, use: foundation of loosely coupled design
DI: most common IoC implementation, class receives dependencies from outside, loosely coupled and testable, use: everywhere in ASP.NET Core
Types: Constructor injection (most common, preferred), Property injection, Interface injection, Service Locator (anti-pattern, avoid)
What is the Interface Segregation Principle (ISP)?
ISP: don't force a class to implement methods it doesn't need, split large interfaces into smaller focused ones, IWorkable + IFeedable instead of one big IWorker, use: keeps classes clean, avoids empty or throw implementations
What is the difference between value types and reference types? How does Stack and Heap relate to them?
Value types: store actual value, copy on assignment, independent memory, stack allocated, int bool struct, use: small simple data
Reference types: store memory address, object lives on heap, multiple variables share same object, changes reflect across all references, class string array, use: complex objects
Stack: stores value types + method call frames, fast LIFO allocation
Heap: stores reference type objects, managed by Garbage Collector
What is boxing and unboxing in C#?
Boxing: value type → object, value copied to heap, implicit, performance overhead, object o = 10;, use: avoid in hot paths, use generics instead
Unboxing: object → value type, explicit cast required, throws if wrong type, int x = (int)o;
What is Garbage Collection in .NET?
GC: automatically frees heap memory of unreferenced objects, no manual memory management needed
Generations: Gen 0 (short-lived, collected frequently), Gen 1 (survived one collection), Gen 2 (long-lived objects)
Note: GC handles managed memory only, implement IDisposable for unmanaged resources like DB connections, file handles
What is IDisposable and the using statement?
IDisposable: interface with Dispose(), implement when class holds unmanaged resources like DB connections, file handles, streams, use: manual resource cleanup
using statement: auto-calls Dispose() when block ends, even on exception, prevents resource leaks, using var conn = new SqlConnection();
What is the difference between Task, Thread, and Async/Await? What is Thread Pool and how does TPL work?
Thread: raw OS thread, heavy, manual management, expensive to create/destroy, use: rarely used directly
Thread Pool: managed pool of reusable threads by .NET, avoids create/destroy overhead, tasks run on pool threads by default
Task: lightweight unit of work on thread pool, supports cancellation and continuation, Task.Run(() => DoWork()), use: CPU-bound background work
TPL: Task Parallel Library, provides Parallel.For Parallel.ForEach Task.WhenAll, use: CPU-heavy parallel workloads
async/await: frees thread while waiting, doesn't block, await GetDataAsync(), use: I/O-bound work like DB/HTTP calls
How does async/await work under the hood? What is ConfigureAwait(false)?
async/await: on hitting await, method pauses and returns control to caller without blocking thread, resumes when task completes, compiler converts to state machine under the hood
ConfigureAwait(false): resume on any thread, not original context, avoids deadlocks, use: library or non-UI code
What is Parallel programming? Difference between Parallel.For, Parallel.ForEach, and Task.WhenAll?
Parallel programming: runs operations simultaneously across multiple CPU cores, use: CPU-bound work only, not I/O
Parallel.For: parallel loop over index range, Parallel.For(0, 10, i => Process(i));, use: fixed range CPU work
Parallel.ForEach: parallel loop over collection, Parallel.ForEach(list, item => Process(item));, use: collection CPU work
Task.WhenAll: runs multiple async tasks concurrently, waits for all, await Task.WhenAll(task1, task2);, use: multiple independent I/O calls
Thread safety: use ConcurrentBag<T> or lock when threads share data
What is the difference between var, dynamic, and object?
var: compile-time type inference, strongly typed, type fixed after assignment, var x = 5 = int x = 5, use: LINQ, complex type names
object: base type of all types, holds anything but needs explicit casting, compile-time checked, use: when type is truly unknown
dynamic: runtime type resolution, no compile-time checking, errors only at runtime, use: COM interop, reflection, avoid in regular code
What is the difference between readonly and const?
const: compile-time constant, assigned at declaration only, implicitly static, primitives/string/enum only, const int Max = 100;, use: fixed known values like PI, config constants
readonly: runtime constant, assigned at declaration or constructor, any type allowed, per-instance or static, readonly int _id;, use: values known only at runtime or per instance
What is the difference between value, ref, and out parameters?
value: copy passed, original untouched, default behavior, use: when caller's variable should not change
ref: passes original variable, must be initialized before passing, changes reflect in caller, ref int x, use: method needs to read and modify caller's variable
out: passes original variable, no need to initialize before passing, method must assign before returning, int.TryParse("5", out int result), use: returning multiple values from a method
What is the difference between == and .Equals()?
==: reference equality by default for objects (same memory address), overridden for strings to compare values, use: primitives and strings
.Equals(): value equality, compares actual content, can be overridden in custom classes, use: comparing object values, custom equality logic
What is the difference between IEnumerable and IQueryable?
IEnumerable: executes query in-memory client side, data already fetched from DB, filtering happens in C#, use: in-memory collections
IQueryable: builds expression tree, executes on DB server side, filters run as SQL, more efficient, use: EF Core DB queries, always prefer over IEnumerable for DB
What is the difference between First, FirstOrDefault, Single, and SingleOrDefault?
First(): returns first match, throws if none found, use: when result is guaranteed to exist
FirstOrDefault(): returns first match, returns null if none, use: safe default choice in most cases
Single(): expects exactly one match, throws if none or more than one, use: unique lookup where duplicates are a bug
SingleOrDefault(): returns null if none, throws if more than one, use: unique lookup that may not exist
What is the difference between Dictionary and HashSet?
Dictionary<K,V>: key-value pairs, lookup value by key, keys must be unique, O(1) lookup, dict["key"] = value;, use: mapping/lookup by key
HashSet<T>: unique values only, no key-value pair, O(1) lookup, set.Contains(x);, use: membership check, removing duplicates
What are Delegates in C#? What is the difference between Action, Func, and Predicate?
Delegate: type-safe function pointer, holds reference to method, supports multicast via +=, use: callbacks, events, LINQ
Action: built-in delegate, returns void, 0–16 params, Action<string> a = msg => Console.WriteLine(msg);, use: operations with no return value
Func: built-in delegate, returns value, last type param is return type, Func<int,int,int> f = (a,b) => a+b;, use: LINQ, transformations
Predicate: delegate returning bool, Predicate<int> p = x => x > 10;, use: filtering/conditions in collections
Event: delegate with safety wrapper, only raised from declaring class, use: publisher-subscriber pattern
What are Generics and why are they useful?
Generics: write type-safe code that works with any type without duplication, no casting needed, no boxing/unboxing overhead, List<T> Dictionary<K,V>, use: reusable collections, repositories, utility methods
What are Extension Methods?
Extension Methods: add methods to existing types without modifying or inheriting them, defined in static class, first param uses this keyword, public static int WordCount(this string s), use: extend third-party or built-in types, all LINQ methods are extension methods
What is the difference between String and StringBuilder?
string: immutable, every modification creates new object in memory, use: few or no modifications
StringBuilder: mutable, modifies same object in place, much faster for repeated changes, sb.Append("text");, use: loops with many string concatenations

SECTION 2. ASP.NET Core / Web API ▼

What is ASP.NET MVC architecture? What are its components?
MVC: separates app into 3 layers, clean separation of concerns
Model: data and business logic, C# classes, EF entities, DTOs
View: UI layer, Razor pages or Angular templates
Controller: handles requests, calls services, returns view or JSON, use: entry point for every HTTP request
What is the difference between Program.cs and Startup.cs?
Startup.cs (.NET 5 and below): two methods — ConfigureServices() for DI registration, Configure() for middleware pipeline
Program.cs (.NET 6+): Startup.cs removed, everything in one file, services via builder.Services.Add...(), middleware via app.Use...(), cleaner minimal hosting model
What is middleware and how does the request pipeline work? What is the difference between app.Use(), app.Run(), and app.Map()?
Middleware: components that handle request and response in a pipeline, each can process request, call next, then process response on way back
app.Use(): calls next middleware, used for most cases, use: logging, auth, exception handling
app.Run(): terminal, stops pipeline, no next called, use: final response handler
app.Map(): branches pipeline based on URL path, use: different handling per route
What are the three service lifetimes — Transient, Scoped, Singleton?
Transient: new instance every time requested, use: lightweight stateless services
Scoped: one instance per HTTP request, use: DbContext, request-specific logic
Singleton: one instance for entire app lifetime, use: caching, logging, configuration services
How does JWT authentication work in Web API?
JWT flow: on login server creates signed token with user claims, client sends token in Authorization header on every request, server validates signature and reads claims, no session stored server-side
Config: AddAuthentication().AddJwtBearer(), protect endpoints with [Authorize]
What is OAuth? What are Access Token and Refresh Token? What is the difference between OAuth and JWT?
OAuth 2.0: authorization framework, lets user grant third-party app access without sharing password, use: Login with Google/Facebook
JWT: token format, compact signed string with claims, OAuth uses JWT as token format but JWT can be used independently too
Access Token: short-lived (minutes/hours), sent in Authorization header, use: access protected APIs
Refresh Token: long-lived, stored in HttpOnly cookie, used to get new access token without re-login, never sent to API directly
How do you secure an API? How do you handle security vulnerabilities?
Auth: JWT with short expiry, [Authorize] on all protected endpoints, HTTPS only
CORS: restrict allowed origins, builder.Services.AddCors() + app.UseCors()
SQL Injection: use parameterized queries or EF Core, never concatenate user input into SQL
XSS: Angular auto-escapes by default, avoid innerHTML with user data
CSRF: use anti-forgery tokens for form submissions
Secrets: store in Azure Key Vault, never in code
Other: input validation, rate limiting, never expose sensitive data in responses
What is CORS and how do you configure it?
CORS: browser security rule, blocks requests from different domain/port, server must explicitly allow origins
Config: builder.Services.AddCors() to define policy, app.UseCors("PolicyName") to apply, use: allow Angular frontend to call API from different port/domain
What is the difference between IActionResult, ActionResult, and specific return types?
IActionResult: most flexible, can return any HTTP response, Ok() NotFound() BadRequest(), use: when return type varies
ActionResult<T>: flexible + typed, better for Swagger docs, use: most common in real projects
Specific type (e.g. User): simplest but loses HTTP status code control, use: simple read-only endpoints
What is the difference between PUT and PATCH? How do you handle PATCH for updating a specific field like email?
PUT: replaces entire resource, send full object, use: full update
PATCH: partial update, send only changed fields, use: update single field like email
PATCH implementation: read only changed field from body, update that property on entity, call SaveChanges(), use JsonPatchDocument<T> for clean implementation
How do you return custom HTTP status codes from a Web API?
Built-in helpers: return Ok(data) (200), return NotFound() (404), return BadRequest("msg") (400), return StatusCode(500, "error") for custom codes, all return IActionResult
What is Idempotency? Why does it matter in APIs?
Idempotent: calling same operation multiple times produces same result as calling once
GET, PUT, DELETE: idempotent — POST: not idempotent, calling twice creates two records
Why it matters: network timeouts cause retries, idempotent endpoints won't create duplicate data
Implementation: idempotency keys — client sends unique request ID, server checks if already processed
How do you perform input validation? What is FluentValidation?
Data Annotations: [Required] [MaxLength] on model, check ModelState.IsValid in controller, use: simple validations
FluentValidation: validation rules in separate class, clean readable syntax, better for complex rules, register with AddFluentValidation() in Program.cs, use: complex conditional validations
How do you implement global exception handling in ASP.NET Core? Why would you still use try-catch inside a controller?
Global handling: custom middleware wrapping pipeline in try-catch, register first so it catches all exceptions, or use built-in app.UseExceptionHandler() or IExceptionFilter
try-catch in controller: use when specific error is expected and needs meaningful handling, e.g. catch DbUpdateException to return clear 400 instead of generic 500
What are Filters in ASP.NET Core? How are they different from middleware?
Filters: run inside MVC pipeline, have access to controller context (action name, model, result)
Types: Authorization (check access first), Action (before/after action), Exception (handle MVC errors), Result (before/after response sent)
vs Middleware: middleware runs before MVC, no controller context, use: filters for MVC-specific logic, middleware for app-wide concerns
What is API caching and how can it be implemented?
Caching: stores response data so server doesn't recompute on every request
Response caching: [ResponseCache] attribute, use: simple endpoint-level caching
In-memory cache: IMemoryCache, use: single server apps
Distributed cache: Redis, use: multi-server/load-balanced setups
What is API versioning and how do you implement it?
API versioning: release API changes without breaking existing clients
Approaches: URL versioning /api/v1/users (most common), header versioning, query string ?api-version=1.0
Implementation: Asp.Versioning.Mvc NuGet package, configure in Program.cs
What is content negotiation in Web API? How do you restrict it to specific formats?
Content negotiation: client requests format via Accept header, server responds accordingly, default is JSON in ASP.NET Core
Restrict to JSON: don't register AddXmlSerializerFormatters(), or use [Produces("application/json")] on specific action
What is state management and data passing in MVC?
HTTP is stateless — ASP.NET Core provides several options:
ViewData/ViewBag: controller to view, single request only
TempData: survives one redirect, uses session under the hood
Session: per user, stored server-side
Query strings / Route values: passed in URL
Cookies: stored in browser, persists across requests
How do you log errors in ASP.NET Core?
Built-in: inject ILogger<T>, use _logger.LogError() LogWarning() LogInformation(), providers: Console, Debug, EventLog
Production: use Serilog (write to files, DB, cloud) or NLog, use: structured logging with Application Insights or Seq
Which unit testing framework do you use and how do you mock data?
Framework: xUnit (preferred in .NET Core) or NUnit
Moq: creates fake versions of dependencies, test class in isolation, var mock = new Mock<IUserRepository>(); then mock.Setup(r => r.GetById(1)).Returns(fakeUser);
use: test only business logic, not the actual DB

SECTION 2.1. Microservices ▼

What are Microservices? How are they different from Monolithic architecture?
Monolithic: single deployable unit, all features tightly coupled, easy to start but hard to scale and maintain as grows, use: small or early-stage apps
Microservices: app split into small independent services, each handles one business capability, deployed/scaled/updated independently, use: large enterprise apps, independent teams
Why not monolith at scale: hard to scale specific parts, one change needs full redeploy, teams block each other, slow releases
How do microservices communicate with each other?
Synchronous (HTTP/REST or gRPC): service calls another and waits, simple but tightly coupled, caller fails if target is down, use: real-time queries needing immediate response
Asynchronous (Message Queue): service publishes message and doesn't wait, fully decoupled, sender doesn't care if receiver is down, Azure Service Bus / RabbitMQ / Kafka, use: events, notifications, background processing
What is an API Gateway? Why is it important?
API Gateway: single entry point for all client requests, clients never call individual services directly, handles routing, auth, rate limiting, load balancing, logging in one place
Examples: AWS API Gateway, Azure API Management, Ocelot (.NET)
use: enforce security in one place, change backend services without affecting clients
What is the Saga Pattern? How do you handle distributed transactions across microservices?
Saga: executes operations step-by-step across multiple services, if step fails previous steps undone via compensation actions, like SQL transaction but for distributed systems, Order → Payment → Inventory
Choreography: services communicate via events, no central controller, loosely coupled but harder to track, use: Azure Service Bus
Orchestration: central orchestrator coordinates each step, easier to follow but orchestrator is a dependency, use: Azure Durable Functions
What is the Circuit Breaker pattern? Why is it important?
Circuit Breaker: stops retrying a failing service, prevents cascading failures across system
States: Closed (normal), Open (failures exceeded threshold, returns fallback immediately), Half-Open (tries one request to check recovery)
Implementation: Polly library in .NET, supports retry, circuit breaker, timeout policies, use: any service-to-service HTTP call
How does each microservice manage its own database?
Database per Service: each service owns its own DB, services never share DB directly, communicate via APIs or events
Benefits: each service picks best DB type, teams stay independent, schema changes don't break others
Downside: joining data across services is harder, use API calls or eventual consistency via events
How do you trace requests across multiple microservices? What did you use in Aviva?
Distributed Tracing: unique Correlation ID generated at entry point, passed in HTTP header to every downstream service, each service logs this ID, filter all logs for single request across all services
Aviva: implemented correlation ID logging across 30+ services, reduced debugging time from 2–3 days to ~2 hours
What is Service Discovery in microservices?
Service Discovery: services find each other automatically without hardcoded URLs, needed because IPs change on dynamic deployments
Client-side: caller queries registry (Consul, Eureka) to get address, then calls directly
Server-side: caller goes through load balancer or API Gateway which handles lookup, use: built-in in AWS/Azure cloud setups
What are the main challenges of microservices and how do you handle them?
Distributed tracing: hard to debug across services → Correlation IDs + Serilog + Application Insights
Data consistency: no shared DB, eventual consistency → Saga pattern
Service failures: one failing service can cascade → Circuit Breaker with Polly, retry with exponential backoff
Network latency: more calls than monolith → async messaging, caching
Deployment complexity: many services to manage → Docker + Kubernetes, CI/CD pipelines

SECTION 3. Entity Framework Core ▼

What is DbContext and DbSet?
DbContext: main class, manages DB connection and all operations (query, save, track changes), one per request (Scoped lifetime)
DbSet<T>: represents a table, each property maps to one table, query through DbSet, commit via SaveChanges()
What is the difference between Code First and Database First?
Code First: write C# model classes, EF generates DB from them, use migrations for schema changes, use: new projects, full control from code
Database First: DB already exists, EF scaffolds C# models from it, use: existing legacy databases
What are EF Core migrations?
Migrations: track model changes over time, apply to DB incrementally, like version control for DB schema
Add-Migration MigrationName creates migration file, Update-Database applies it, use: every time model/schema changes
What is the difference between Lazy Loading and Eager Loading?
Eager Loading: loads related data immediately with .Include(), single query with JOIN, use: when you know you need related data
Lazy Loading: related data loads automatically when navigation property accessed, fires separate query per access, causes N+1 problem, use: avoid in most cases
What is the N+1 query problem? How do you fix it?
N+1 problem: load 100 orders then access order.Customer in loop — fires 101 queries total, one per row
Fix: use .Include(o => o.Customer) — loads everything in one JOIN query upfront, use: always include related data you know you'll need
What is AsNoTracking() and when should you use it?
AsNoTracking(): skips change tracking for fetched entities, faster queries, less memory, EF doesn't monitor those objects, context.Users.AsNoTracking().ToList(), use: read-only queries where you don't plan to update data
How do you optimize EF Core queries?
AsNoTracking(): for all read-only queries
.Select(): fetch only needed columns, avoid loading full entity
.Include(): avoid N+1, load related data upfront
IQueryable: keep query as IQueryable as long as possible, filters run as SQL not in memory
Pagination: use Skip/Take, never load all rows
Indexes: add proper DB indexes on filtered/joined columns
What is the difference between Add, Update, and Attach in EF Core?
Add: marks entity as new, INSERT on SaveChanges, use: inserting new records
Update: marks all properties as modified, UPDATE on SaveChanges even for unchanged fields, use: full entity update
Attach: attaches existing entity without marking modified, manually set changed properties, use: partial update of specific fields only
How do you configure a Many-to-Many relationship in EF Core?
EF Core 5+ (direct): no junction entity needed, modelBuilder.Entity<Student>().HasMany(s => s.Courses).WithMany(c => c.Students)
With junction table: needed when extra columns required (e.g. enrollment date), create StudentCourse entity with composite key StudentId + CourseId, configure two one-to-many relationships

SECTION 4. Design Patterns & Architecture ▼

What are the SOLID principles?
S — Single Responsibility: class should have only one reason to change, one job only, UserService handles user logic — not emails too
O — Open/Closed: open for extension, closed for modification, add new behavior via new class not by changing existing code
L — Liskov Substitution: child class should be replaceable for parent without breaking app, Penguin should not inherit Bird.Fly() since penguins can't fly
I — Interface Segregation: don't force class to implement methods it doesn't need, split large interfaces into smaller focused ones
D — Dependency Inversion: high-level classes depend on abstractions not concrete implementations, Controller depends on IUserService not UserService
Which design patterns have you used? Explain each.
Singleton: one shared instance throughout app, use: config service, logging, caching
Repository: abstracts DB access from business logic, easier to test and swap DBs, use: data access layer
Factory: creates objects without exposing creation logic, use: when object creation is complex or conditional
Strategy: swap algorithms at runtime without changing client code, use: payment methods, sorting strategies
Decorator: add behavior to objects dynamically without modifying them, use: logging, caching wrappers
What is the Repository pattern? What is a Generic Repository?
Repository: wraps data access logic, controllers/services never talk to DbContext directly, easier to mock in unit tests, easier to swap DB
Generic Repository: common CRUD methods for any entity type using generics, GetById GetAll Add Delete, less code duplication, use: base repository all entities inherit from
What is the CQRS pattern?
CQRS: separates read and write operations into different models
Command: write side, changes state (Create, Update, Delete), returns no data
Query: read side, fetches data, never changes state
Why: reads and writes scale differently, reads can be optimized separately with caching or denormalized models
Implementation: MediatR library in .NET, use: microservices, event-sourced systems
Explain your project architecture — layers and responsibilities.
Layered architecture: Presentation (Controllers / Angular) → Business Logic (Services) → Data Access (Repositories / EF Core) → Database
each layer talks only to layer below it, concerns separated, code testable, easy to maintain, use: standard enterprise app structure
How does Angular communicate with Web API? Walk through a request flow.
Flow: Angular service calls HttpClient.get() → HTTP request hits API controller → controller calls service → service calls repository → EF Core queries DB → data returns up the chain → Angular receives JSON → component displays it
JWT: attached to every request via HTTP Interceptor automatically
How do you improve performance in your APIs? How would you design a scalable API?
EF queries: AsNoTracking() for reads, .Select() for only needed columns, pagination with Skip/Take
Caching: IMemoryCache for single server, Redis for multi-server, reduces DB load
Async/await: everywhere for non-blocking I/O, frees threads during DB/HTTP calls
Scalable design: stateless (no server-side session), load balancer for horizontal scaling, message queue for heavy background tasks, rate limiting
DB: proper indexes on filtered/joined columns
How do you handle large data queries efficiently?
Pagination: always use Skip/Take, never load all rows at once
.Select(): fetch only needed columns, avoid full entity load
Streaming: use AsAsyncEnumerable() for bulk exports, streams results instead of loading all into memory
Indexes: add on filtered columns, avoid client-side filtering
How do you handle concurrency issues?
Optimistic Concurrency: add RowVersion or Timestamp column to entity, EF Core checks on update — if changed since last read throws DbUpdateConcurrencyException, catch and inform user record was already modified, use: when multiple users can edit same record
How do you handle exceptions across the entire project? How do you log and monitor errors in production?
Backend: global middleware catches all unhandled exceptions, returns standard error response, log everything via Serilog
Angular: HTTP Interceptor catches error responses, shows toast/notification
Expected errors: return proper HTTP status codes (400, 404, 409)
Production monitoring: Serilog → Azure Application Insights or Seq, set up alerts for error spikes and failed requests
How do you deploy your application? Walk through your deployment process.
Build: ng build --prod for Angular, dotnet publish for .NET API
CI/CD: push to Git → Azure DevOps / GitHub Actions runs tests, builds artifacts, deploys to Azure App Service

What are SQL Constraints? What is the difference between a primary key and a unique key?
Constraints: rules enforced on columns to maintain data integrity
PRIMARY KEY: unique, not null, only one per table, main row identifier
UNIQUE: no duplicates, allows one NULL, multiple allowed per table
FOREIGN KEY: links to primary key of another table, enforces referential integrity
NOT NULL: column cannot be empty
CHECK: value must satisfy condition, CHECK (Age >= 18)
DEFAULT: sets default value if none provided
Explain all types of SQL joins.
INNER JOIN: rows matching in both tables only
LEFT JOIN: all rows from left + matching from right, null if no match, use: most common, keep all left rows
RIGHT JOIN: all from right + matching from left
FULL JOIN: all rows from both tables, nulls where no match
CROSS JOIN: every row from A × every row from B, cartesian product
SELF JOIN: table joined with itself, use: employee-manager hierarchy in same table
What is the difference between WHERE and HAVING?
WHERE: filters rows before grouping, works on regular columns
HAVING: filters groups after GROUP BY, works on aggregates like COUNT SUM
Rule: regular column filter = WHERE, aggregate filter = HAVING
What are transactions in SQL? Explain ACID.
Transaction: group of operations that either all succeed or all fail together
Atomicity: all or nothing
Consistency: DB stays in valid state before and after
Isolation: concurrent transactions don't interfere with each other
Durability: committed data survives crashes
What are deadlocks in SQL? How do you prevent them?
Deadlock: two transactions wait for each other to release a lock, neither can proceed, SQL Server auto-detects and kills one
Prevention: always access tables in same order, keep transactions short, use READ COMMITTED SNAPSHOT isolation to reduce locking conflicts
How do you implement exception handling inside a SQL Server stored procedure?
TRY/CATCH: wrap logic in BEGIN TRY...END TRY / BEGIN CATCH...END CATCH
In CATCH: read error via ERROR_MESSAGE() ERROR_NUMBER(), rollback via ROLLBACK, re-throw or log
use: always wrap transactions in TRY/CATCH so failures don't leave data in broken state
What are indexes? How do they help and what are the downsides?
Indexes: let DB find rows fast without full table scan, like a book index, huge speedup for SELECT
Downside: INSERT/UPDATE/DELETE slower (index must update too), extra storage
use: add on columns used in WHERE, JOIN, ORDER BY — not on every column
How do you optimize SQL queries?
Indexes: on frequently filtered/joined columns
Avoid SELECT *: fetch only needed columns
Avoid functions on indexed columns in WHERE: defeats the index
JOINs over subqueries: generally better performance
Pagination: OFFSET/FETCH for large results
Execution Plan: always check to find actual bottleneck
What is the difference between ROW_NUMBER, RANK, and DENSE_RANK?
ROW_NUMBER: always unique sequential number, no ties (1,2,3,4), use: pagination, picking one row per group
RANK: ties get same number, next rank skips (1,2,2,4), use: leaderboards with gaps
DENSE_RANK: ties get same number, no gaps (1,2,2,3), use: rankings where continuous numbering matters
What is the difference between DELETE, TRUNCATE, and DROP?
DELETE: removes specific rows with WHERE, fully logged, triggers fire, can rollback, use: conditional row removal
TRUNCATE: removes all rows fast, minimal logging, resets identity column, can't rollback, use: clear entire table quickly
DROP: removes entire table (structure + data), permanent, use: delete table completely
What is the difference between a stored procedure and a user-defined function?
Stored Procedure: can SELECT, INSERT, UPDATE, DELETE, can have output params, cannot be used inside SELECT, use: business logic operations
Function: must return a value, used inside SELECT/WHERE, cannot modify data, use: reusable calculations
What is a CTE? Difference between CTE, Temp Table, and Table Variable?
CTE: named temporary result set with WITH, exists for single query only, no physical storage, no indexes, use: readable one-time queries, recursive queries like hierarchy trees

String Problems

1. Reverse a string in place.
Convert to char array, swap from both ends moving inward. 
string Reverse(string s)
{
    char[] chars = s.ToCharArray();
    int left = 0, right = chars.Length - 1;
    while (left < right)
    {
        (chars[left], chars[right]) = (chars[right], chars[left]);
        left++; right--;
    }
    return new string(chars);
}
2. Reverse the words in a sentence.
Split by space, reverse the array, join back. 
string ReverseWords(string s)
{
    string[] words = s.Split(' ');
    Array.Reverse(words);
    return string.Join(' ', words);
}
3. Check if a string is a palindrome.
Compare from both ends — if any characters don't match, it's not a palindrome. 
bool IsPalindrome(string s)
{
    s = s.ToLower();
    int left = 0, right = s.Length - 1;
    while (left < right)
    {
        if (s[left] != s[right]) return false;
        left++; right--;
    }
    return true;
}
4. Count the occurrences of a given word in a string.
Split by space, count how many words match the target. 
int CountOccurrences(string sentence, string word)
{

    return words.Count(w => w.Equals(word, StringComparison.OrdinalIgnoreCase));
}
5. Find the first non-repeated character in a string.
Count frequency of each character, return the first one with count 1. 
char? FirstNonRepeated(string s)
{
    var freq = new Dictionary<char, int>();
    foreach (char c in s)
        freq[c] = freq.GetValueOrDefault(c) + 1;

    foreach (char c in s)
        if (freq[c] == 1) return c;

    return null;
}

Sort both strings — if equal, they are anagrams. 
bool AreAnagrams(string a, string b)
{
    if (a.Length != b.Length) return false;
    return a.OrderBy(c => c).SequenceEqual(b.OrderBy(c => c));
}

14. Reorder array: even numbers first, then divisible by 3, then remaining odd numbers (no duplicates, maintain order).
Add evens first, then div-by-3 (not already added), then remaining odds. Use HashSet to track added numbers. 
// Input:  { -6, 5, -1, 3, 0, 10, -3, 9 }
// Output: -6, 0, 10, 3, -3, 9, 5, -1

int[] ReorderArray(int[] arr)
{
    var seen = new HashSet<int>();
    var result = new List<int>();

    // Step 1: even numbers
    foreach (var n in arr)
        if (n % 2 == 0 && seen.Add(n))
            result.Add(n);

    // Step 2: divisible by 3 (not already added)
    foreach (var n in arr)
        if (n % 3 == 0 && seen.Add(n))
            result.Add(n);

    // Step 3: remaining odd numbers
    foreach (var n in arr)
        if (seen.Add(n))
            result.Add(n);

    return result.ToArray();
}

        a = b;
        b = temp;
    }
    return b;
}

30. Write a thread-safe Singleton class in C#.
Use lock with double-null check — thread safe, creates instance only when first accessed. 
public sealed class Singleton
{
    private static Singleton _instance;
    private static readonly object _lock = new object();

    private Singleton() { }

    public static Singleton Instance
    {

        {

                return _instance;
            }
        }
    }
