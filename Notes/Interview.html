SECTION 1. C# ▼

What is the difference between value types and reference types? How does Stack and Heap relate to them?
Value types (int, bool, struct) store the actual value and are usually allocated on the Stack. Copying creates a new separate value.
Reference types (class, string, array, object) store the object in the Heap, while the variable on the Stack holds only a reference to it.
Stack is fast and used for short-lived data, while Heap stores objects and is managed by the Garbage Collector.
What is boxing and unboxing in C#?
Boxing = Converting a value type into reference type (object type, From stack to the heap). Unboxing = pulling it back out.
Example: object o = 5; is boxing, int i = (int)o; is unboxing. Avoid it in loops, creates unnecessary heap allocations & slows things down.
What is the difference between == and .Equals()?
For objects, == checks if both variables point to the same memory address (reference equality). .Equals() checks if the values are the same (value equality). For strings, == is overridden to compare values, so both behave the same for strings.
What is the difference between readonly and const?
const is set at compile time — it never changes and must be a simple type like int or string. readonly is set at runtime, usually in the constructor, so each instance can have a different value. Use readonly when the value isn't known until the app runs.
What is the difference between value, ref, and out parameters?
value = a copy is passed, original is untouched. ref = passes the original variable, must be initialized before passing. out = passes the original variable, doesn't need to be initialized — but the method must assign it before returning.
What is the difference between String and StringBuilder?
string is immutable — every time you modify it, a new string object is created in memory. StringBuilder modifies the same object in place. If you're doing lots of string concatenation (like in a loop), always use StringBuilder — it's much faster and uses less memory.
What is the difference between var, dynamic, and object?
var — type inferred at compile time, still strongly typed. var x = 5 is just int x = 5 to the compiler.

object — base type of everything, can hold any value but requires casting. Type-checked at compile time.

dynamic — type resolved at runtime, no compile-time checking. Useful for COM interop or unknown types — avoid in regular code, errors only show at runtime.
What is the difference between a concrete class and an abstract class?
A concrete class is fully implemented and can be instantiated directly. An abstract class cannot be instantiated — it's a base class that can have both implemented and abstract (unimplemented) methods. Use it when you want to share common logic but force child classes to provide their own version of certain methods.
What is the difference between an interface and an abstract class? Can a class implement multiple interfaces? What if two interfaces have the same method name?
Interface — pure contract, no implementation (before C# 8), multiple allowed.
Abstract class — can have both implemented + abstract methods, only one allowed.
Use interface for unrelated classes sharing behavior; abstract class when classes also share code.

Multiple interfaces — yes, a class can implement many interfaces.

Same method name conflict — if same signature, one shared implementation satisfies both. If different behavior needed, use explicit interface implementation:
void IAnimal.Speak() { ... } and void IHuman.Speak() { ... } — called through the specific interface reference.
Types of constructors? What is a private constructor? How many static constructors can a class have?
Types: Default, Parameterized, Copy, Static, Private. A private constructor prevents other classes from creating an instance — used in the Singleton pattern. A class can only have one static constructor (no parameters allowed), and it runs automatically once before the class is used for the first time.
What is IDisposable and the using statement?
Implement IDisposable when your class holds unmanaged resources like DB connections, file handles, or streams. The using block automatically calls Dispose() when the block ends — even if an exception is thrown. This prevents resource leaks.
What are Delegates in C#? What is the difference between Action, Func, and Predicate?
Delegate — type-safe pointer to a method. Pass methods around like variables.

Action — no return value. Action<string> print = msg => Console.WriteLine(msg);
Func<T> — returns a value. Last type param is return type. Func<int, int, int> add = (a, b) => a + b;
Predicate<T> — returns bool, used for filtering. Predicate<int> isEven = n => n % 2 == 0;

Multicast — delegate can point to multiple methods using +=, all called in order.
Event — delegate with extra safety, can only be raised from the declaring class. Used for publisher-subscriber patterns (e.g. button click).
What is Parallel programming? Difference between Parallel.For, Parallel.ForEach, and Task.WhenAll?
Runs operations simultaneously across multiple CPU cores. Part of TPL. Use for CPU-bound work — not I/O (use async/await for that).

Parallel.For — parallel for loop across index range. Parallel.For(0, 10, i => Process(i));
Parallel.ForEach — parallel foreach over a collection. Parallel.ForEach(list, item => Process(item));
Task.WhenAll — runs multiple async tasks concurrently, waits for all. Best for I/O. await Task.WhenAll(GetOrdersAsync(), GetUsersAsync());

Thread safety — when threads share data, use ConcurrentBag<T> or lock to avoid race conditions.
What are Extension Methods?
They let you add new methods to an existing type without modifying it or inheriting from it. Defined in a static class, first parameter uses the this keyword. Example: public static int WordCount(this string s). LINQ methods like .Where() and .Select() are all extension methods.
What are Generics and why are they useful?
Generics let you write code that works with any type without duplicating it. Example: List<T> works for int, string, or any object. Benefits: type safety (no casting), code reuse, and better performance (no boxing/unboxing).
What is the difference between IEnumerable and IQueryable?
IEnumerable runs queries in-memory (client side) — data is already fetched from DB. IQueryable builds an expression tree and runs the query on the DB (server side). Always prefer IQueryable with EF Core so filters like .Where() run as SQL, not after loading all rows.
What is the difference between First, FirstOrDefault, Single, and SingleOrDefault?
First() — returns first match, throws if none found.
FirstOrDefault() — returns first match, returns null if none. Prefer this in most cases.

Single() — expects exactly one match, throws if none or more than one.
SingleOrDefault() — returns null if none, still throws if more than one.

Use Single when exactly one result is guaranteed (e.g., find by unique ID).
What is the difference between Dictionary and HashSet?
Dictionary<K,V> — key-value pairs, look up value by key, keys must be unique.
HashSet<T> — unique values only, no key-value, use for membership check or removing duplicates.

Both use hashing internally — lookup is O(1).
What is the difference between Task, Thread, and Async/Await? What is Thread Pool and how does TPL work?
Thread — raw OS thread, heavy, manual management.
Thread Pool — managed pool of reusable threads maintained by .NET — avoids overhead of creating/destroying threads constantly. Tasks run on thread pool threads by default.

Task — lighter unit of work on the thread pool, supports cancellation and continuation.
TPL (Task Parallel Library) — built on Tasks, provides Parallel.For, Parallel.ForEach, Task.WhenAll to run work across multiple cores. Use for CPU-heavy parallel workloads.

async/await — syntactic sugar over Task, makes async code look synchronous. Use for I/O (DB, HTTP) — frees the thread while waiting instead of blocking it.
How does async/await work under the hood? What is ConfigureAwait(false)?
When you hit await, the method pauses and returns control to the caller without blocking the thread. When the awaited task completes, execution resumes — by default on the original thread (e.g., UI thread). ConfigureAwait(false) tells it not to resume on the original thread — use this in library code to avoid deadlocks.
What is Garbage Collection in .NET?
The GC automatically frees memory of objects that are no longer referenced. It works in 3 generations: Gen 0 (short-lived), Gen 1 (survived one collection), Gen 2 (long-lived). You don't manually free memory in .NET, but implement IDisposable for unmanaged resources — GC doesn't handle those.
What is Dependency Injection (DI)? What is Inversion of Control (IoC)? What are the types?
IoC — principle where control of dependencies is inverted to an external system instead of the class itself.
DI — most common IoC implementation. Class receives dependencies from outside instead of creating them — loosely coupled and testable.

Types: Constructor injection (most common), Setter/Property injection, Interface injection, Service Locator (anti-pattern — avoid it).
What is the Interface Segregation Principle (ISP)?
Don't force a class to implement methods it doesn't need. Instead of one big interface, split it into smaller focused ones. Example: Instead of one IWorker with Work() and Eat(), split into IWorkable and IFeedable so a Robot only implements IWorkable.
What is the Singleton pattern? How do you make it thread-safe?
Singleton ensures only one instance of a class exists throughout the app. Thread-safe way: use Lazy<T> — private static readonly Lazy<T> instance = new(() => new T());. Lazy<T> handles thread safety automatically and only creates the instance when first accessed.

SECTION 2. ASP.NET Core / Web API ▼

What is ASP.NET MVC architecture? What are its components?
MVC separates the application into 3 layers:

Model — data and business logic, plain C# classes, EF entities, DTOs.
View — UI layer, Razor pages or Angular templates that display model data.
Controller — handles requests, calls services/models, returns view or JSON.

UI changes don't affect business logic and vice versa — clean separation of concerns.
What are the three service lifetimes — Transient, Scoped, Singleton?
Transient: new instance every time it's requested — use for lightweight stateless services. Scoped: one instance per HTTP request — use for DbContext. Singleton: one instance for the entire app lifetime — use for caching, logging, configuration services.
What is the difference between Program.cs and Startup.cs?
Startup.cs (.NET 5 and below) — had two methods: ConfigureServices() to register DI services and Configure() to set up middleware pipeline.

Program.cs (.NET 6+) — Startup.cs removed, everything merged into one file using minimal hosting model. Services via builder.Services.Add...(), middleware via app.Use...() — cleaner and simpler.
What is middleware and how does the request pipeline work? What is the difference between app.Use(), app.Run(), and app.Map()?
Middleware are components that handle requests and responses in a pipeline. Each one processes the request, optionally calls the next middleware, then processes the response on the way back.

app.Use() — calls next middleware, used for most cases.
app.Run() — terminal, stops the pipeline, no next called.
app.Map() — branches the pipeline based on URL path.
How do you implement global exception handling in ASP.NET Core?
Create a custom middleware that wraps the entire pipeline in a try-catch and returns a standard error response. Register it first so it catches exceptions from all other middleware. Alternatively use the built-in app.UseExceptionHandler() or implement IExceptionFilter.
What is the difference between IActionResult, ActionResult, and specific return types?
IActionResult is the most flexible interface — can return Ok, NotFound, BadRequest, etc. ActionResult<T> gives you flexibility plus type info (better for Swagger). Specific return types like User are simplest but you lose control over HTTP status codes.
What is the difference between PUT and PATCH? How do you handle PATCH for updating a specific field like email?
PUT — replaces entire resource, send full object.
PATCH — partial update, send only changed fields.

Email update via PATCH — send { "email": "new@email.com" } only. In controller, read the field, update only that property on entity, call SaveChanges().
For clean implementation use JsonPatchDocument<T> from Microsoft.AspNetCore.JsonPatch.
How do you return custom HTTP status codes from a Web API?
Use built-in helper methods: return Ok(data) (200), return NotFound() (404), return BadRequest("msg") (400), return StatusCode(500, "error") for anything custom. All of these return IActionResult.
How do you perform input validation? What is FluentValidation?
Basic: add Data Annotations on the model ([Required], [MaxLength]) and check ModelState.IsValid in the controller. FluentValidation is a library where you write validation rules in a separate class with clean, readable syntax — better for complex rules. Wire it up with AddFluentValidation() in Program.cs.
Why would you still use try-catch inside a controller action?
Global error handling catches unexpected exceptions, but sometimes you need specific handling — like catching a DbUpdateException to return a clear message instead of a generic 500. Use try-catch when an error is expected and you want to handle it meaningfully in that specific context.
What are Filters in ASP.NET Core? How are they different from middleware?
Filters run inside the MVC pipeline and have access to controller context (action name, model, result). Types: Authorization (check access first), Action (before/after action method), Exception (handle MVC exceptions), Result (before/after response is sent). Middleware runs before MVC and has no controller context.
How does JWT authentication work in Web API?
On login, the server creates a signed JWT token with user claims and sends it to the client. On every future request, the client sends this token in the Authorization header. The server validates the signature and reads the claims — no session stored server-side. Configure with AddAuthentication().AddJwtBearer().
What is OAuth? What are Access Token and Refresh Token? What is the difference between OAuth and JWT?
OAuth 2.0 — authorization framework, lets a user grant a third-party app access to their resources without sharing password. Example: "Login with Google".

JWT — token format (compact signed string with claims). OAuth uses JWT as the token format, but JWT can be used independently too.

Access Token — short-lived (minutes/hours), used to access protected APIs, sent in Authorization header.
Refresh Token — long-lived, used to get a new Access Token when it expires without re-login. Stored in HttpOnly cookie, never sent to the API directly.
What is CORS and how do you configure it?
CORS is a browser security rule that blocks requests from a different domain/port. You need to tell your API which origins are allowed. In ASP.NET Core: define a policy with builder.Services.AddCors(), then apply it with app.UseCors("PolicyName").
How do you secure an API?
JWT for authentication, [Authorize] on endpoints, HTTPS only, CORS policy to restrict origins, input validation to prevent injection, rate limiting to prevent abuse, and never expose sensitive data in responses. For server-to-server calls, use API keys.
What is Idempotency? Why does it matter in APIs?
Idempotent — calling the same operation multiple times produces the same result as calling it once.

GET, PUT, DELETE — idempotent. POST — not idempotent, calling twice creates two records.

Important for reliability — if a network timeout causes a retry, idempotent endpoints won't create duplicate data.
Implement with idempotency keys — client sends a unique request ID, server checks if already processed.
How do you handle security vulnerabilities in your application?
SQL Injection — use parameterized queries or EF Core, never concatenate user input into SQL.
XSS — Angular auto-escapes output by default, avoid innerHTML with user data.
CSRF — use anti-forgery tokens for form submissions.
Sensitive data — store secrets in Azure Key Vault, never in code, use HTTPS always.
Auth — JWT with short expiry, validate all inputs, [Authorize] on all protected endpoints, CORS to restrict origins.
What is API versioning and how do you implement it?
Versioning lets you release API changes without breaking existing clients. Common approaches: URL versioning (/api/v1/users), header versioning, or query string (?api-version=1.0). Use the Asp.Versioning.Mvc NuGet package and configure it in Program.cs.
What is API caching and how can it be implemented?
Caching stores response data so the server doesn't recompute it on every request. Options: Response caching with the [ResponseCache] attribute, In-memory cache with IMemoryCache, or Distributed cache with Redis for multi-server setups.
How do you log errors in ASP.NET Core?
Inject ILogger<T> and use _logger.LogError(), LogWarning(), LogInformation(). Built-in providers: Console, Debug, EventLog. For production use Serilog (write to files, DB, cloud) or NLog.
What is content negotiation in Web API? How do you restrict it to specific formats?
Content negotiation is when the client tells the server what format it wants (JSON, XML) via the Accept header, and the server responds accordingly. By default ASP.NET Core returns JSON. To restrict to JSON only, call .AddControllers().AddJsonOptions() and remove the XML formatter — or simply don't register AddXmlSerializerFormatters() in program.cs file.
For a specific action, use this attribute [Produces("application/json")].
What is state management and data passing in MVC?
HTTP is stateless, so ASP.NET Core gives you several ways to pass data: ViewData/ViewBag (controller to view, single request), TempData (survives one redirect, uses session under the hood), Session (per user, stored server-side), Query strings / Route values (in the URL), and Cookies (stored in browser).
Which unit testing framework do you use and how do you mock data?
Most common framework is xUnit (preferred in .NET Core projects) or NUnit. For mocking, use Moq — it lets you create fake versions of dependencies so you can test a class in isolation. Example: mock your repository interface with var mock = new Mock<IUserRepository>();, set up what it returns with mock.Setup(r => r.GetById(1)).Returns(fakeUser);, then inject it into the class you're testing. This way you test only your logic, not the DB.

SECTION 2.1. Microservices ▼

What are Microservices? How are they different from Monolithic architecture?
Monolithic: entire application is one deployable unit — all features tightly coupled. Easy to develop initially but hard to scale and maintain as it grows. Microservices: application is split into small independent services — each handles one business capability (orders, payments, users). Each can be deployed, scaled, and updated independently. Aviva is a real example — 30+ independent services (policy, claims, documents) each deployed separately.
How do microservices communicate with each other?
Synchronous (HTTP/REST or gRPC): service calls another directly and waits for response — simple but tightly coupled, if one service is down the caller fails too. Asynchronous (Message Queue): service publishes a message and doesn't wait — done via Azure Service Bus, RabbitMQ, or Kafka. Services are decoupled — sender doesn't care if receiver is down.
What is an API Gateway? Why is it important?
A single entry point for all client requests — clients never call individual services directly. It handles routing, authentication, rate limiting, load balancing, and logging in one place. Examples: AWS API Gateway, Azure API Management, Ocelot (for .NET). You enforce security in one place and can change backend services without affecting clients.
How do you handle distributed transactions across microservices?
You can't use a single DB transaction across services. The common pattern is Saga.
Choreography Saga: each service publishes events and reacts to others' events — no central coordinator. Loosely coupled but harder to track.
Orchestration Saga: a central orchestrator tells each service what to do step by step. Easier to follow but orchestrator becomes a dependency.
If a step fails, compensating transactions (rollback actions) undo previous steps.
What is the Circuit Breaker pattern? Why is it important?
When one service keeps failing, dependant services shouldn't keep retrying — it wastes resources and cascades failures across the system.
Circuit Breaker monitors failures. After a threshold is hit, it opens (stops sending requests) and returns a fallback immediately. After a timeout it goes half-open — tries one request to check recovery. If it succeeds, it closes again.
In .NET, implement with Polly library — supports retry, circuit breaker, and timeout policies.
How do you trace requests across multiple microservices? What did you use in Aviva?
Use Distributed Tracing with a Correlation ID — a unique ID generated at the entry point and passed in the HTTP header to every downstream service. Each service logs this ID so you can filter all logs for a single request across all services.
At Aviva, I implemented this solution which reduced debugging time from 2–3 days to about 2 hours across 30+ services.
How does each microservice manage its own database?
Each microservice owns its own database — called the Database per Service pattern. Services never share a DB directly; they communicate through APIs or events.
Benefits: each service picks the best DB type for its needs, teams stay independent, schema changes in one service don't break others. Downside: joining data across services is harder — you call APIs or use eventual consistency through events.
What is Service Discovery in microservices?
Services are dynamically deployed — their IP addresses change. Service Discovery lets services find each other automatically without hardcoded URLs.
Client-side: caller queries a registry like Consul or Eureka to get the address, then calls directly. Server-side: caller goes through a load balancer or API Gateway which handles the lookup. In cloud setups (AWS/Azure) this is usually built in.
What are the main challenges of microservices and how do you handle them?
Distributed tracing — hard to debug across services. Solution: Correlation IDs + centralized logging (Serilog + Application Insights).
Data consistency — no shared DB, eventual consistency. Solution: Saga pattern.
Service failures — one failing service can cascade. Solution: Circuit Breaker with Polly, retries with exponential backoff.
Network latency — more network calls than monolith. Solution: async messaging where possible, caching.
Deployment complexity — many services to manage. Solution: Docker + Kubernetes, CI/CD pipelines.

SECTION 3. Entity Framework Core ▼

What is DbContext and DbSet?
DbContext is the main class that manages the DB connection and all operations (query, save, track changes). DbSet<T> represents a table — each property in your context maps to one table. You query through DbSet and call SaveChanges() on the context to commit.
What is the difference between Code First and Database First?
Code First: write C# model classes, EF generates the database from them. Database First: database already exists, EF scaffolds the C# models from it. Code First is preferred for new projects since you control everything from code and use migrations for changes.
What are EF Core migrations?
Migrations track changes to your model over time and apply them to the database incrementally. Add-Migration MigrationName creates the migration file. Update-Database applies it. Think of it like version control for your database schema.
What is the difference between Lazy Loading and Eager Loading?
Eager Loading: load related data immediately with .Include() — one query with a JOIN. Lazy Loading: related data loads automatically when you access the navigation property — fires a separate query. Prefer Eager Loading to avoid the N+1 problem.
What is AsNoTracking() and when should you use it?
By default, EF tracks every entity you fetch and watches for changes. AsNoTracking() skips that — EF doesn't monitor those objects. Use it for read-only queries (like displaying a list) where you don't plan to update anything. It's faster and uses less memory.
What is the difference between Add, Update, and Attach in EF Core?
Add = marks entity as new (INSERT on save). Update = marks all properties as modified (UPDATE on save — even unchanged ones). Attach = attaches an existing entity without marking it modified — use when you only want to update specific properties manually.
What is the N+1 query problem? How do you fix it?
If you load 100 orders and then access order.Customer in a loop, EF fires a separate query for each — that's 101 queries total. Fix it with .Include(o => o.Customer) — loads everything in one JOIN query upfront.
How do you configure a Many-to-Many relationship in EF Core?
EF Core 5+ — supports many-to-many directly without a junction entity:
modelBuilder.Entity<Student>().HasMany(s => s.Courses).WithMany(c => c.Students)

With junction table (if you need extra columns like enrollment date) — create a StudentCourse entity with StudentId and CourseId as a composite key, then configure two one-to-many relationships.

In SQL: junction table holds the foreign keys of both entities — each row represents one relationship.
How do you optimize EF Core queries?
Use AsNoTracking() for reads, .Select() to fetch only needed columns, .Include() to avoid N+1, keep queries as IQueryable as long as possible before calling ToList(), use pagination with Skip/Take, and add proper indexes on the DB.

SECTION 4. Design Patterns & Architecture ▼

Which design patterns have you used? Explain each.
Singleton: one shared instance (e.g., config service). Repository: abstracts DB access logic from business logic. Factory: creates objects without exposing creation logic. Strategy: swap algorithms at runtime. Decorator: add behavior to objects dynamically. Always mention ones you actually used in your project with a real example.
What is the Repository pattern? What is a Generic Repository?
Repository wraps data access logic so controllers/services don't talk to DbContext directly — easier to test (you can mock the repo) and easier to swap DBs. A Generic Repository provides common CRUD methods (GetById, GetAll, Add, Delete) for any entity type using generics — less code duplication.
What is the CQRS pattern?
CQRS (Command Query Responsibility Segregation) — separates read and write operations into different models.

Command — write side, changes state (Create, Update, Delete), returns no data.
Query — read side, fetches data, never changes state.

Why — read and write workloads scale differently. Reads can be optimized separately (e.g., denormalized read models, caching). Common in microservices and event-sourced systems.
In .NET, typically implemented with MediatR library.
Explain your project architecture — layers and responsibilities.
Typical layered setup: Presentation (Controllers / Angular) → Business Logic (Services) → Data Access (Repositories / EF Core) → Database. Each layer talks only to the one below it. This keeps concerns separated, code testable, and easy to maintain.
How does Angular communicate with Web API? Walk through a request flow.
Angular service calls HttpClient.get() → HTTP request hits API controller → controller calls service → service calls repository → EF Core queries DB → data returns back up the chain → Angular receives JSON → component displays it. JWT token is attached to every request via an HTTP Interceptor.
How do you handle exceptions across the entire project?
Backend: global middleware catches unhandled exceptions and returns a standard error response. Angular: HTTP Interceptor catches error responses and shows a toast/notification. For expected errors (validation, not found), return proper HTTP status codes. Log everything on the backend with Serilog.
How do you improve performance in your APIs?
Use AsNoTracking() and .Select() in EF queries, caching for frequently read data (IMemoryCache / Redis), pagination to avoid loading huge datasets, async/await everywhere for non-blocking I/O, proper DB indexes, and avoid unnecessary object allocations.
How do you handle concurrency issues?
Use Optimistic Concurrency: add a RowVersion or Timestamp column to the entity. EF Core checks this on update — if it changed since you read it, it throws a DbUpdateConcurrencyException. Catch it and tell the user the record was already modified by someone else.
How do you handle large data queries efficiently?
Never load all rows at once — use pagination (Skip/Take). Use .Select() to fetch only the columns you need. For exports or bulk reads, use AsAsyncEnumerable() to stream results instead of loading everything into memory. Add proper DB indexes and avoid client-side filtering.
How would you design a scalable API?
Use stateless design (no server-side session) so any instance can handle any request. Add caching (Redis) to reduce DB load. Use async/await throughout for non-blocking I/O. Implement rate limiting. Use a message queue (like Azure Service Bus) for heavy background tasks. Deploy behind a load balancer so you can scale horizontally.
How do you deploy your application? Walk through your deployment process.
Build the Angular app (ng build --prod) and the .NET API (dotnet publish). Push code to a repo (Git), CI/CD pipeline (Azure DevOps / GitHub Actions) runs tests, builds the artifacts, and deploys to the server or Azure App Service. EF migrations run as part of the deploy script. Environment configs come from App Settings / environment variables — never hardcoded.
How do you log and monitor errors in production?
Use Serilog to write structured logs to a sink like Azure Application Insights, Seq, or a log file. Log all unhandled exceptions via global middleware. Set up alerts on Application Insights for error spikes or failed requests. For the Angular side, use an error interceptor to report frontend errors to the backend or a monitoring service.
What are the SOLID principles?
SOLID is a set of 5 design principles that make code cleaner, more maintainable, and easier to extend.

S — Single Responsibility Principle
A class should have only one reason to change. Each class should do one job only.
Example: A UserService should handle user logic — not also send emails.

O — Open/Closed Principle
A class should be open for extension but closed for modification.
Example: Add new behaviour by creating a new class, not by changing existing code.

L — Liskov Substitution Principle
A child class should be replaceable for its parent class without breaking the app.
Example: If Bird has a Fly() method, a Penguin class should not inherit it since penguins cannot fly.

I — Interface Segregation Principle
Do not force a class to implement methods it does not need. Split large interfaces into smaller focused ones.

D — Dependency Inversion Principle
High-level classes should not depend on low-level classes. Both should depend on abstractions (interfaces).
Example: A Controller should depend on IUserService, not directly on UserService.

SECTION 5. Angular ▼

What are the main building blocks of an Angular application?
Modules: group related components/services. Components: UI building blocks (template + logic). Services: shared business logic and data. Directives: manipulate the DOM. Pipes: transform data in templates. Routing: navigate between views. Everything is wired together through modules and DI.
What are Angular services? How are they different from components?
Services handle business logic, HTTP calls, and shared state — they have no UI. Components handle the view and UI. Services are singletons (by default) shared across the app. You inject a service into a component via the constructor — the component just calls it and displays the result.
What is Dependency Injection in Angular and how does it work?
Angular has a built-in DI system. When a component or service needs a dependency, Angular's injector looks it up in the provider registry and supplies it automatically. You register services with providedIn: 'root' (app-wide singleton) or in a specific module/component. No need to manually create instances.
What is the Angular component lifecycle? Main hooks?
ngOnChanges: runs when @Input values change. ngOnInit: runs once after init — best place for API calls. ngDoCheck: runs on every change detection cycle. ngOnDestroy: runs before removal — use to unsubscribe from observables and clean up resources.
What is the difference between constructor and ngOnInit?
Constructor runs first and is only for dependency injection. ngOnInit runs after Angular sets up the component and its inputs are available. Always put initialization logic and API calls in ngOnInit — at constructor time, the component isn't fully ready yet.
What is @Input() and @Output()? How does parent-child communication work?
@Input(): parent passes data down to child via property binding. @Output(): child sends events up to parent using EventEmitter. Parent → child = Input. Child → parent = Output + emit. For unrelated components, use a shared service or state management.
What is ViewChild and ContentChild?
@ViewChild: access a child component or DOM element in your own template. @ContentChild: access content projected into your component via <ng-content>. Use ViewChild when you need to call a child component's method or directly interact with a DOM element.
What is ng-content and content projection in Angular?
Content projection lets a parent pass HTML content into a child component's template. The child uses <ng-content> as a placeholder — whatever the parent puts inside the child's tags gets rendered there. It's how you build reusable wrapper components like modals, cards, or panels that accept custom inner content.
What are directives? Structural vs Attribute directives?
Structural directives change the DOM structure — add/remove elements. Examples: *ngIf, *ngFor. Attribute directives change the appearance or behavior of an existing element. Examples: ngClass, ngStyle. Components are technically directives with a template.
What is trackBy in *ngFor and why is it important?
By default, when an array changes Angular re-renders all DOM elements. trackBy tells Angular to identify items by a unique key (like id), so it only re-renders items that actually changed. Big performance improvement for large lists.
What are pipes? Pure vs Impure? What is data transformation in Angular?
Pipes — transform data directly in templates. Example: {{ date | date:'short' }}, {{ name | uppercase }}.
This is Angular's built-in data transformation — format, filter, or convert values without touching component logic.

Pure pipes — only re-run when input reference changes. Fast.
Impure pipes — re-run on every change detection cycle. Slow — use carefully.
Create custom pipes with the @Pipe decorator.
What is an Auth Guard (Route Guard) in Angular?
A guard that protects routes from unauthorized access. Implement CanActivate — if user isn't logged in, redirect to login page. Apply in routing: { path: 'dashboard', canActivate: [AuthGuard] }. CanDeactivate can warn users before leaving a form with unsaved changes.
What is Lazy Loading in Angular?
Instead of loading all modules upfront, lazy loading loads them only when the user navigates to that route. Reduces initial bundle size and speeds up startup. Set up with loadChildren: () => import('./feature/feature.module').then(m => m.FeatureModule) in routing config.
How does routing work in Angular? What are RouterModule, router-outlet, and RouterLink?
RouterModule is the module that enables routing — you define your routes in it. <router-outlet> is the placeholder in the template where the matched component is rendered. RouterLink is the directive you use instead of <a href> to navigate without full page reloads.
What is the difference between RouterModule.forRoot() and RouterModule.forChild()?
forRoot() is used once in the root AppModule — it sets up the router and registers the main routes. forChild() is used in feature modules — it registers additional routes without re-initializing the router. Using forRoot() in a feature module causes bugs.
Template-driven vs Reactive forms? Where have you used forms in your project?
Template-driven — form logic in HTML with ngModel, simple, less code, hard to test.
Reactive — form logic in component class with FormGroup and FormControl, fully testable, dynamic, better for complex forms. Prefer Reactive in real projects.

In Aviva — used Reactive Forms for the policy update and claims submission screens. Had complex validations (conditional required fields, custom validators for date ranges) — Reactive forms made it easy to control validation programmatically and test it with unit tests.
How do you make HTTP calls in Angular? What is HttpClientModule?
Import HttpClientModule in your AppModule, then inject HttpClient into your service. Use this.http.get<T>(url), .post(), .put(), .delete() — all return Observables. Always make HTTP calls inside a service, not directly in components.
What is an HTTP Interceptor?
Interceptors sit between your HTTP calls and the server — they can modify every request or response. Common uses: attach JWT token to every request, show a loading spinner, handle 401/403 errors globally, log all requests. Implement HttpInterceptor and register it as a provider.
How do you use the response of one service call in another service call in Angular?
Use switchMap — chain the second call after the first using RxJS:

this.userService.getUser(id).pipe(switchMap(user => this.orderService.getOrders(user.id))).subscribe(...)

switchMap — cancels previous inner observable, use when second call depends on first result.
mergeMap — runs both in parallel, use for independent calls.
Avoid nested subscribe() calls — leads to callback hell and memory leaks.
What is RxJS? Observable vs Promise?
RxJS is a library for reactive programming with streams. Observable: lazy, can emit multiple values over time, can be cancelled, supports operators (map, filter, switchMap). Promise: eager, resolves once, can't cancel. Use Observables in Angular for HTTP calls (cancellable) and real-time streams.
What are Promises in JavaScript?
Promise — represents a future value, either resolved or rejected.
3 states: pending → fulfilled or rejected.

.then() — handles success. .catch() — handles error. .finally() — runs always.

Eager — starts immediately when created. Resolves only once. Can't cancel.
Use async/await for cleaner syntax over raw .then() chains.
What is the difference between Subject and BehaviorSubject?
Subject: emits values only to current subscribers — new subscribers miss previous values. BehaviorSubject: holds the latest value and emits it immediately to any new subscriber. Use BehaviorSubject for shared state like the current logged-in user — any component subscribing gets the latest value right away.
What is the difference between switchMap, mergeMap, and concatMap?
switchMap: cancels the previous request and starts a new one — best for search (only care about latest result). mergeMap: runs all requests in parallel — use for independent operations. concatMap: waits for each to finish before starting next — use when order matters.
What is the async pipe and why is it preferred?
The async pipe subscribes to an Observable in the template and automatically unsubscribes when the component is destroyed — no memory leaks. You don't need to manually call subscribe() and unsubscribe(). Cleaner code and safer by default.
What is Change Detection? Default vs OnPush?
Change detection is how Angular checks if the UI needs to update. Default: checks the entire component tree on every event — safe but can be slow. OnPush: only re-checks when an @Input reference changes or an event fires inside the component. Use OnPush for better performance in heavy components.
How do you share data and manage state between unrelated components?
Use a shared service with a BehaviorSubject — any component can update the value and any other component subscribed to it will get the update. For larger apps, use a state management library like NgRx (Redux pattern) which gives you a single store, actions, reducers, and selectors for predictable state.
How do you make an Angular application responsive?
CSS Flexbox / Grid — use for layout, adapts to screen size naturally.
Angular CDK BreakpointObserver — detect screen size in component and conditionally show/hide elements.
Media queries — in component CSS or global styles.
Angular Material — built-in responsive grid and components.
ngClass / ngStyle — apply classes dynamically based on screen size or component state.
Avoid fixed pixel widths — use %, vw, rem instead.
How do you optimize the performance of an Angular application?
Use OnPush change detection, lazy loading for feature modules, trackBy in *ngFor, the async pipe to avoid manual subscriptions, pure pipes instead of methods in templates, preload strategies for routes, and run ng build --prod which enables AOT compilation, tree shaking, and minification.
What are Angular Migrations? How do you upgrade an Angular app to a newer version?
Upgrading Angular from one version to another using Angular CLI.
Main command: ng update @angular/core @angular/cli — updates packages and runs migration schematics (auto fixes).
Rule: Upgrade one major version at a time. Check update.angular.io for breaking changes.
Flow: Check version → Update CLI → Run ng update → Fix issues → Test app.

SECTION 6. SQL & Database ▼

What is the difference between a primary key and a unique key?
Both enforce uniqueness, but Primary Key cannot be NULL and there can only be one per table. Unique Key allows one NULL and a table can have multiple unique keys. Primary key is the main row identifier; unique key just prevents duplicate values on a column.
What are SQL Constraints?
Rules enforced on table columns to maintain data integrity.

PRIMARY KEY — unique, not null, one per table.
FOREIGN KEY — links to primary key of another table, enforces referential integrity.
UNIQUE — no duplicate values, allows one NULL.
NOT NULL — column cannot be empty.
CHECK — value must satisfy a condition. Example: CHECK (Age >= 18).
DEFAULT — sets a default value if none provided.
What is the difference between WHERE and HAVING?
WHERE filters rows before grouping. HAVING filters groups after GROUP BY. Simple rule: filtering on a regular column = WHERE. Filtering on an aggregate like COUNT or SUM = HAVING.
Explain all types of SQL joins.
INNER JOIN: rows matching in both tables. LEFT JOIN: all rows from left + matching from right (null if no match). RIGHT JOIN: all from right + matching from left. FULL JOIN: all rows from both, nulls where no match. CROSS JOIN: every row from A × every row from B. SELF JOIN: join a table with itself (e.g., employee-manager in the same table).
What is the difference between ROW_NUMBER, RANK, and DENSE_RANK?
All assign a number within a partition. ROW_NUMBER: always unique (1,2,3,4) even for ties. RANK: ties get the same number but next rank skips (1,2,2,4). DENSE_RANK: ties get the same number, no gaps (1,2,2,3). Use DENSE_RANK when you need continuous rankings.
What is the difference between DELETE, TRUNCATE, and DROP?
DELETE: removes specific rows (with WHERE), fully logged, triggers fire, can rollback. TRUNCATE: removes all rows fast, minimal logging, resets identity column, can't rollback in most DBs. DROP: removes the entire table — structure and data gone permanently.
What is the difference between a stored procedure and a user-defined function?
Stored Procedure: can do anything (SELECT, INSERT, UPDATE, DELETE), can have output params, cannot be used inside a SELECT. Function: must return a value, can be used inside SELECT/WHERE, but cannot modify data. Use functions for reusable calculations; use procedures for business logic operations.
What is a CTE? What is the difference between CTE and Temp Table? Can you create indexes on a CTE?
CTE (Common Table Expression) — named temporary result set defined with WITH. Exists only for that single query, no physical storage.

CTE vs Temp Table:
CTE — no storage, no index, single query use, better readability. Good for recursive queries (e.g., hierarchy trees).
Temp Table (#TempTable) — physically stored in tempdb, persists for the session, can have indexes, reusable across multiple queries.

Use CTE for one-time readable queries. Use Temp Table when reusing result multiple times or need indexes for performance.

Indexes on CTE — No, you cannot create indexes on a CTE directly. If you need indexes, use a Temp Table instead.
What is the difference between Temp Table and Table Variable?
Temp Table (#TempTable) — stored in tempdb, supports indexes, visible within the session, can be used across stored procedures.

Table Variable (@TableVar) — stored in memory (mostly), scoped to the current batch/procedure only, no indexes except primary key, better for small datasets.

Use Temp Table for large data sets or when you need indexes. Use Table Variable for small, short-lived datasets.
How do you implement exception handling inside a SQL Server stored procedure?
Use a BEGIN TRY ... END TRY / BEGIN CATCH ... END CATCH block. Inside the CATCH block you can read the error with ERROR_MESSAGE(), ERROR_NUMBER(), roll back the transaction with ROLLBACK, and re-throw or log the error. Always wrap transactions inside TRY/CATCH so a failure doesn't leave data in a broken state.
What are transactions in SQL? Explain ACID.
A transaction is a group of operations that either all succeed or all fail together. Atomicity = all or nothing. Consistency = DB stays in a valid state. Isolation = concurrent transactions don't interfere with each other. Durability = committed data survives crashes.
What are deadlocks in SQL? How do you prevent them?
A deadlock is when two transactions wait for each other to release a lock — neither can proceed. SQL Server auto-detects and kills one. Prevention: always access tables in the same order, keep transactions short, and use READ COMMITTED SNAPSHOT isolation to reduce locking conflicts.
What are indexes? How do they help and what are the downsides?
Indexes let the DB find rows fast without scanning every row — like a book's index. Huge speedup for SELECT queries. Downside: INSERT/UPDATE/DELETE become slower (index must be updated too), and they use extra storage. Add indexes on columns used in WHERE, JOIN, and ORDER BY — not on every column.
How do you optimize SQL queries?
Add indexes on frequently filtered/joined columns, avoid SELECT *, avoid using functions on indexed columns in WHERE (defeats the index), use JOINs over subqueries where possible, paginate large results with OFFSET/FETCH, and check the Execution Plan to find the actual bottleneck.

SECTION 9. Coding Questions ▼

String Problems

1. Reverse a string in place.
Convert to char array, swap from both ends moving inward. 
string Reverse(string s)
{
    char[] chars = s.ToCharArray();
    int left = 0, right = chars.Length - 1;
    while (left < right)
    {
        (chars[left], chars[right]) = (chars[right], chars[left]);
        left++; right--;
    }
    return new string(chars);
}
2. Reverse the words in a sentence.
Split by space, reverse the array, join back. 
string ReverseWords(string s)
{
    string[] words = s.Split(' ');
    Array.Reverse(words);
    return string.Join(' ', words);
}
3. Check if a string is a palindrome.
Compare from both ends — if any characters don't match, it's not a palindrome. 
bool IsPalindrome(string s)
{
    s = s.ToLower();
    int left = 0, right = s.Length - 1;
    while (left < right)
    {
        if (s[left] != s[right]) return false;
        left++; right--;
    }
    return true;
}
4. Count the occurrences of a given word in a string.
Split by space, count how many words match the target. 
int CountOccurrences(string sentence, string word)
{
    string[] words = sentence.Split(' ');
    return words.Count(w => w.Equals(word, StringComparison.OrdinalIgnoreCase));
}
5. Find the first non-repeated character in a string.
Count frequency of each character, return the first one with count 1. 
char? FirstNonRepeated(string s)
{
    var freq = new Dictionary<char, int>();
    foreach (char c in s)
        freq[c] = freq.GetValueOrDefault(c) + 1;

    foreach (char c in s)
        if (freq[c] == 1) return c;

    return null;
}
6. Check if a string contains only digits.
Use char.IsDigit on every character — all must be digits. 
bool IsAllDigits(string s) => s.All(char.IsDigit);
7. Check if two strings are anagrams of each other.
Sort both strings — if equal, they are anagrams. 
bool AreAnagrams(string a, string b)
{
    if (a.Length != b.Length) return false;
    return a.OrderBy(c => c).SequenceEqual(b.OrderBy(c => c));
}

8. Remove duplicates from an array.

int[] RemoveDuplicates(int[] arr) => arr.Distinct().ToArray();
9. Find the second largest number in an array.
Get distinct values, order descending, skip first, take next. 
int SecondLargest(int[] arr) =>

10. Find all duplicate elements in an array.
Group by value, return groups with count greater than 1. 
int[] FindDuplicates(int[] arr)
{
    return arr.GroupBy(x => x)
              .Where(g => g.Count() > 1)
              .Select(g => g.Key)
              .ToArray();
}
11. Find the missing number in an array (1 to N).
Expected sum of 1 to N minus actual sum gives the missing number. 
int FindMissing(int[] arr, int n)
{
    int expected = n * (n + 1) / 2;
    return expected - arr.Sum();
}
12. Sort an array without using any built-in sort method.
Bubble sort — compare adjacent elements and swap if out of order. 
void BubbleSort(int[] arr)
{
    for (int i = 0; i < arr.Length - 1; i++)
        for (int j = 0; j < arr.Length - i - 1; j++)
            if (arr[j] > arr[j + 1])
                (arr[j], arr[j + 1]) = (arr[j + 1], arr[j]);
}
13. Return numbers divisible by 2 and numbers divisible by 3.
Filter array twice using LINQ — one for each condition. 
void DivisibleNumbers(int[] arr)
{
    int[] byTwo   = arr.Where(x => x % 2 == 0).ToArray();
    int[] byThree = arr.Where(x => x % 3 == 0).ToArray();

    Console.WriteLine("Divisible by 2: " + string.Join(", ", byTwo));
    Console.WriteLine("Divisible by 3: " + string.Join(", ", byThree));
}
14. Reorder array: even numbers first, then divisible by 3, then remaining odd numbers (no duplicates, maintain order).
Add evens first, then div-by-3 (not already added), then remaining odds. Use HashSet to track added numbers. 
// Input:  { -6, 5, -1, 3, 0, 10, -3, 9 }
// Output: -6, 0, 10, 3, -3, 9, 5, -1

int[] ReorderArray(int[] arr)
{
    var seen = new HashSet<int>();
    var result = new List<int>();

    // Step 1: even numbers
    foreach (var n in arr)
        if (n % 2 == 0 && seen.Add(n))
            result.Add(n);

    // Step 2: divisible by 3 (not already added)
    foreach (var n in arr)
        if (n % 3 == 0 && seen.Add(n))
            result.Add(n);

    // Step 3: remaining odd numbers
    foreach (var n in arr)
        if (seen.Add(n))
            result.Add(n);

    return result.ToArray();
}

Number Problems

15. Return the sum of the digits of an integer.
Convert to string, parse each character as digit and sum. 
int SumOfDigits(int n)
{
    return Math.Abs(n).ToString().Sum(c => c - '0');
}
16. Calculate the factorial of a number using recursion.
Base case is 0 or 1 = 1. Each call multiplies n by factorial of n-1. 
int Factorial(int n)
{
    if (n <= 1) return 1;
    return n * Factorial(n - 1);
}
17. Return the N-th number in the Fibonacci sequence.
Each number is the sum of the previous two. Base cases: 0 and 1. 
int Fibonacci(int n)
{
    if (n <= 1) return n;
    int a = 0, b = 1;
    for (int i = 2; i <= n; i++)
    {
        int temp = a + b;
        a = b;
        b = temp;
    }
    return b;
}
18. Swap two variables without using a third variable.
Use tuple deconstruction — cleanest way in C#. Or use XOR / arithmetic for integers. 
// Method 1: Tuple (preferred in C#)
int a = 5, b = 10;
(a, b) = (b, a);

// Method 2: Arithmetic
a = a + b;  // a = 15
b = a - b;  // b = 5
a = a - b;  // a = 10

// Method 3: XOR (integers only)
a = a ^ b;
b = a ^ b;
a = a ^ b;

23. Rank employees by salary within each department.
Use DENSE_RANK with PARTITION BY to rank within each department separately. 
SELECT Name, DepartmentId, Salary,
    DENSE_RANK() OVER (PARTITION BY DepartmentId ORDER BY Salary DESC) AS Rnk
FROM Employees;
24. Write a query to get the top 3 salaries in each department.
Rank salaries per department, then filter where rank is 3 or less. 
SELECT Name, DepartmentId, Salary, Rnk
FROM (
    SELECT Name, DepartmentId, Salary,
        DENSE_RANK() OVER (PARTITION BY DepartmentId ORDER BY Salary DESC) AS Rnk
    FROM Employees
) ranked
WHERE Rnk <= 3;
25. Write a query to implement pagination (page 2, 10 records per page).
Use OFFSET to skip rows and FETCH NEXT to take the page size. 
SELECT * FROM Employees
ORDER BY EmployeeId
OFFSET 10 ROWS          -- skip first 10 (page 1)
FETCH NEXT 10 ROWS ONLY; -- take next 10 (page 2)
26. Write a query to find duplicate emails in a table.
GROUP BY email, use HAVING to find groups with more than one row. 
SELECT Email, COUNT(*) AS Count
FROM Employees
GROUP BY Email
HAVING COUNT(*) > 1;
27. Write a CTE to get a running total of sales by date.
Use SUM with OVER and ORDER BY — window function accumulates row by row. 
SELECT SaleDate, Amount,
    SUM(Amount) OVER (ORDER BY SaleDate) AS RunningTotal
FROM Sales
ORDER BY SaleDate;
28. Write a query to filter and search with dynamic conditions.
Use WHERE with optional conditions — pass NULL to skip a filter. 
SELECT * FROM Employees
WHERE
    (@DepartmentId IS NULL OR DepartmentId = @DepartmentId)
    AND (@Name IS NULL OR Name LIKE '%' + @Name + '%')
    AND (@MinSalary IS NULL OR Salary >= @MinSalary)
ORDER BY Name
OFFSET @Skip ROWS FETCH NEXT @Take ROWS ONLY;

29. Write a custom middleware in ASP.NET Core.
Implement InvokeAsync — process request, call next, process response. 
public class LoggingMiddleware
{
    private readonly RequestDelegate _next;

    public LoggingMiddleware(RequestDelegate next) => _next = next;

    public async Task InvokeAsync(HttpContext context)
    {
        Console.WriteLine($"Request: {context.Request.Path}");
        await _next(context);
        Console.WriteLine($"Response: {context.Response.StatusCode}");
    }
}

// Register in Program.cs
app.UseMiddleware<LoggingMiddleware>();
30. Write a thread-safe Singleton class in C#.
Use lock with double-null check — thread safe, creates instance only when first accessed. 
public sealed class Singleton
{
    private static Singleton _instance;
    private static readonly object _lock = new object();

    private Singleton() { }

    public static Singleton Instance
    {
        get
        {
            lock (_lock)
            {
                if (_instance == null)
                    _instance = new Singleton();

                return _instance;
            }
        }
    }
}
31. Implement multiple interfaces with the same method signature.
Use explicit interface implementation — each interface gets its own method body, called through its specific interface reference. 
interface IAnimal { void Speak(); }
interface IHuman  { void Speak(); }

class Person : IAnimal, IHuman
{
    void IAnimal.Speak() => Console.WriteLine("Animal sound");
    void IHuman.Speak()  => Console.WriteLine("Hello!");
}

// Usage
Person p = new Person();
((IAnimal)p).Speak(); // "Animal sound"
((IHuman)p).Speak();  // "Hello!"
