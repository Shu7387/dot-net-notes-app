5. JavaScript

SECTION 1: Fundamentals

  • JavaScript: Scripting language used to make web pages interactive & dynamic.
  • Primitive types: Primitives copy actual value, string, null, number, boolean
  • Reference types: Reference types copy address object, array, function and value is on heap
  • == → compares value only, === → compares value & type (strict equality)
  • Truthy vs Falsy values: false, 0, "", null, undefined, NaN Rest all truthy
  • typeof operator: Returns the data type of a value
  • Type casting: Implicit (automatic), Explicit → Manual conversion using Number(), parseInt(), String(), toString()
  • Hoisting: Moving declarations to the top of their scope before code execution
  • var → function-scoped; redeclare & reassign allowed, let → block-scoped; no redeclare, reassign allowed, const → block-scoped; no redeclare & no reassign

SECTION 2: Functions

Topic What it is Why / Where Used
Function Declaration vs Arrow Function Normal functions have their own this(From caller object) and are hoisted; arrow functions don't have their own this(From parent scope) and are not hoisted Normal → object methods, constructors function

SECTION 3: Arrays

  • map: Transform array elements arr.map(x => x * 2)
  • filter: Filter elements by condition arr.filter(x => x > 10)
  • reduce: Reduce to single value arr.reduce((sum, x) => sum + x, 0)
  • forEach: Iterate without return arr.forEach(x => console.log(x))
  • find & findIndex: First matching element & index arr.find(x => x > 10)
  • some & every: At least one match & all match arr.some(x => x > 10)
  • push, pop, shift, unshift, splice, slice: Add/remove array elements arr.splice(startIndex, deleteCount, newItem)

SECTION 4: Objects

  • Object creation: Created using object literals, constructor functions, classes, Object.create, factory functions.
  • Dot notation vs Bracket notation: Dot is for static key obj.name, Bracket is for dynamic key obj[keyName]
  • Object.keys(), Object.values(), Object.entries(): Get object details as collection
  • Destructuring: Extract values from objects or arrays into variables const {name, age} = person
  • Spread operator: Copies or merges values of arrays/objects {...obj}

SECTION 5: Asynchronous JavaScript

  • Synchronous vs Asyn: Sync blocks thread until task complete, async runs task in background without blocking thread.
  • Callback hell: Callback hell is deeply nested callbacks that reduce readability; Use promise or async/await
  • Promises: Handles asnc operation with new Promise((resolve, reject) => {}), .then(), .catch(), .finally(); Promise.Resolve(10)
  • Promise.all(): Wait for multiple promises to resolved
  • async/await: async/await simplifies promises;
  • Event Loop concept: JavaScript executes synchronous code first, then microtasks (Promises / async-await), and finally macrotasks (timers).

SECTION 6: Fetch API & HTTP

  • fetch(): Making HTTP requests
  • GET requests: fetch(url).then(res => res.json())
  • POST requests: fetch(url, {method: 'POST', body: JSON.stringify(data)})
  • Headers: Content-Type, Authorization
  • Response methods: .json(), .text(), .blob()
  • Error handling: fetch rejects promise & throw exception on Network errors, But not on HTTP errors; Must check response.ok
  • async/await with fetch: async/await makes fetch code readable and easy to handle errors

SECTION 7: DOM Manipulation

  • Selecting element: querySelector(), querySelectorAll(), getElementById()
  • Modifying content: innerHTML, textContent, innerText
  • CSS classes: classList.add(), classList.remove(), classList.toggle()
  • Attributes: getAttribute(), setAttribute()
  • Creating/Removing element: createElement(), appendChild(), remove()
  • Styles: element.style.property element.style.color = "red"

SECTION 8: Events

  • addEventListener() & removeEventListener()
  • Common events: click, submit, change, input, keydown, load
  • Event object: event.target, event.preventDefault(), event.stopPropagation()
  • Event delegation: Attaching single event to parent instead of having many in children. It's example of event bubbling.
  • Event bubbling: Default, event moves from child to parent. Button is clicked first, then UI handles it (used for menus, lists, tables, dynamic buttons).
  • Event capturing: Event moves from parent to child. UI checks first, then allows the button click (used for validation, blocking clicks, analytics, modals).

SECTION 9: Miscellaneous

  • JSON.parse(): JSON string to JS Object
  • JSON.stringify(): Opposite of above
  • localStorage: Permanent browser string based storage; localStorage.setItem("Key", "value"), getItem(), removeItem(), clear()
  • setTimeout(() => console.log("Once"), 1000) runs once after delay, setInterval() runs repeatedly
  • Template literals: Makes string concatenation easier, more readable using ${} syntax; Hello ${name}
  • Optional chaining: Safe property access obj?.property
  • Nullish coalescing: Default value for null or undefined value ?? defaultValue
  • Try catch finally: Graceful error handling
